# プロジェクト概要：Unityデスクトップマスコット「Yuukei」

## 1. コア要件とアーキテクチャ
* 動作環境: Windows, Android, macOS, iOS, Linux。Unityベース。現在はWindows版の開発に注力しています。
* アプリ性質: 常駐型デスクトップマスコット。Win32 API（`User32.dll`）を用いてタスクバーおよびAlt+Tabから完全に隠蔽。
* ライフサイクル: メインスレッドとは独立した別スレッドで `System.Windows.Forms.NotifyIcon` のメッセージループを回し、システムトレイ（タスクトレイ）からアプリの操作・終了を行う。
* ウィンドウ制御: `UniWindowController` 等を利用した背景透過、最前面表示、クリックスルー。

## 2. MOD・拡張エコシステムの設計思想
「ユーザー自己責任（セキュリティ含む）」を大前提とした、Minecraftライクな高い自由度を持つ2層構造のMODシステムを構築する。

### A. ライト層向け拡張（スキニング・UI）
* 要件: Unityエディタをインストールさせず、テキストと画像だけでUIやテーマを変更可能にする。
* 技術選定: UI Toolkitを採用。実行時にJSONをパースし、DOMライクにUI要素を動的構築してルートUIツリーにインジェクトするデータ駆動設計。

### B. ヘビー層向け拡張（ロジック・アセット）
* 要件: 全く新しい機能の追加、外部アプリ連携、カスタムオブジェクトの追加を許可する。
* 技術選定: カスタムC# DLLの動的ロード（`Assembly.Load`）およびAssetBundle。
* API設計: 本体側とは疎結合な `Yuukei.ModAPI.dll`（インターフェース群）を定義し、MOD側はこれを参照する。本体起動時にDLLを走査・初期化するプラグインアーキテクチャ。
* イベント管理: 既存の独自DSL（台本データ）のフロー内に、MOD側からC#ロジックによるカスタムコマンドを動的にフック・登録できる仕組みを持たせる。

## 3. 現在の開発状況と方針
* 開発初期段階。タスクバー隠蔽とシステムトレイ基盤は実装済み。
* 現在は、MODから容易に拡張タブやUI要素を差し込めるよう、UI Toolkitをベースとした「設定画面の基盤アーキテクチャ」を構築中。
