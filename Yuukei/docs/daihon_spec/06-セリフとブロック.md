# 6. セリフと処理ブロック

> 条件を満たした際のセリフや動作の記述方法は、1 行で書く場合と複数行で書く場合でルールが異なります。

---

## 6.1. 文（ステートメント）の種類と 1 行 1 文ルール

台本の各行には以下のいずれかの「文」を記述します。**1 行に記述できる文は原則 1 つです。** ただし、セリフと関数呼び出しの連続（表示列）のみ例外的に 1 行に複数記述できます。

| 文の種類 | 1行に複数 | 無条件の例 | 条件付き（1行記法）の例 |
|----------|----------|-----------|----------------------|
| 表示列（セリフ・関数の連続） | 可 | `「こんにちは」＜笑顔＞` | `※（好感度 10 以上）＜笑顔＞` |
| ジャンプ | 不可 | `→おわり` | `※（好感度 0 以下）→おわり` |
| 代入 | 不可 | `好感度=10` | `※（好感度 10 以上）チョコあげた=はい` |
| ブロック開始 | 不可 | （なし） | `※（条件）:` ... `おわり` |

### 表示列

セリフ（`「」`）と関数呼び出し（`＜＞`）は「表示に関連するアクション」として、1 行に自由に連続して記述できます。

```daihon
「こんにちは」「さようなら」
＜不安＞＜w3＞＜笑顔＞
「こんにちは」＜笑顔＞「さようなら」
＜笑顔＞「おはよう」
```

以下は 1 行 1 文ルール違反のためパースエラーになります。

```daihon
好感度=10 夜のあいさつ=「こんばんは」  $$ NG: 代入が 2 つ
好感度=10 ＜笑顔＞                     $$ NG: 代入と関数呼び出しの混在
好感度=10 「ありがとう」                $$ NG: 代入とセリフの混在
```

### 条件付き 1 行記法

`※（条件）` または `※（条件）なら` に続けてアクションを **1 つだけ** 記述する構文です。許可されるアクションは以下の通りです。

- セリフ 1 つ
- 関数呼び出し 1 つ
- 代入 1 つ
- ジャンプ 1 つ
- ブロック開始（`:`）

```daihon
※（好感度 10 以上）なら「好き！」        $$ OK
※（好感度 10 以上）＜笑顔＞              $$ OK
※（好感度 10 以上）なら チョコあげた=はい  $$ OK
※（好感度 0 以下）→おわり                $$ OK
```

以下は 1 行記法のルール違反のためパースエラーになります。

```daihon
※（好感度 10 以上）＜笑顔＞＜w1＞           $$ NG: 関数が 2 つ
※（好感度 10 以上）「こんにちは」「さよなら」 $$ NG: セリフが 2 つ
※（好感度 10 以上）「こんにちは」＜笑顔＞    $$ NG: セリフと関数の混在
```

複数アクションを条件付きで実行する場合は、ブロック記法（§6.2）を使用してください。

> **注意:** セリフ内部の関数呼び出し（例: `「＜笑顔＞おはよう」`）はセリフの構成要素であり、独立したアクションではありません（§6.4 参照）。そのため `※（条件）「＜笑顔＞おはよう」` は 1 行記法として有効です。

---

## 6.2. 複数行記法（ブロック）

1 つの条件に対して複数のセリフや動作をまとめて実行したい場合に使います。条件の後に `:` を置き、次の行から処理を書き、`おわり` で閉じます。

```
※（条件）:
    処理1
    処理2
おわり
```

`なら` を添えることもできます。

```
※（条件）なら：
    処理１
    処理２
おわり
```

```daihon
※（15:00~18:30）なら:
    「これ、朝に言っていたやつ！」
    「チョコあげるね！」
おわり
```

ブロック内ではセリフ、代入、関数呼び出し、ジャンプ（`→`）、ネストされた条件式のすべてが使用可能です。

**空のブロックも許可されます。**

```daihon
※（好感度 50 以上）:
おわり
```

## 6.3. Else If / Else 構文

条件ブロック `※（条件）:` の後に **`※あるいは（条件）:`** で else if チェーン、**`※それ以外:`** で else（フォールバック）を構成できます。

**else if / else はブロック記法専用です。** 1 行記法では使用できません。

### 基本構文

```
※（条件1）なら:
    条件1 が真のときの処理
※あるいは（条件2）なら:
    条件1 が偽かつ条件2 が真のときの処理
※それ以外:
    すべて偽のときの処理
おわり
```

`おわり` はチェーン全体の末尾に 1 つだけ記述し、すべてのブランチを閉じます。個々のブランチ間に `おわり` は不要です。

- else if ブランチは **0 個以上** 記述可能
- `※それ以外:` は **省略可能**

### 例: if + else

```daihon
※（好感度 50 以上）:
    「大好き！」
※それ以外:
    「ふーん」
おわり
```

### 例: if + else if + else

```daihon
※（_今日の運勢 80 以上）:
    ＜笑顔＞
    「すごい！＜_今日の運勢＞点だよ！超ラッキーだね！」
※あるいは（_今日の運勢 30~79）:
    「結果は＜_今日の運勢＞点。まあまあ普通かな？」
※それ以外:
    ＜落ち込んだ顔＞
    「あちゃー……＜_今日の運勢＞点。今日は大人しくしてようね」
おわり
```

### 例: if + else if（else なし）

```daihon
※（天気=「晴れ」）:
    「お出かけしよう！」
※あるいは（天気=「曇り」）:
    「まあ悪くないね」
おわり
```

### ネスト例

```daihon
※（好感度 50 以上）:
    ※（天気=「晴れ」）:
        「最高の日だね！」
    ※あるいは（天気=「曇り」）:
        「まあいい日だね」
    ※それ以外:
        「雨でも楽しいよ！」
    おわり
※それ以外:
    「ふーん」
おわり
```

### 制約

- `※それ以外:` は必ずチェーンの最後に配置する（後に `※（条件）:` を続けるとパースエラー）。
- `※（条件）:` なしで `※それ以外:` を単独使用するとパースエラー。

**実行時の評価:** else if チェーンは上から順に条件を評価し、**最初に真となったブランチのみ**を実行します。一度真のブランチが見つかると、残りの条件は評価されません。

## 6.4. セリフの内部構造

セリフ（`「」` で囲まれた部分）は、プレーンテキストと関数呼び出し `＜＞` の混合です。複数行にまたがることができ、セリフ内の改行はそのまま保持されます。**空のセリフ（`「」`）も許可されます。**

```daihon
「＜笑顔＞おはよう！＜w1＞今日もいい天気だね」
```

上記のセリフは以下のように解析されます。

1. 関数呼び出し: `＜笑顔＞`
2. プレーンテキスト: `おはよう！`
3. 関数呼び出し: `＜w1＞`
4. プレーンテキスト: `今日もいい天気だね`

### 複数行のセリフ

```daihon
「こんにちは、
お元気ですか？」
```

改行を含むセリフとして解析され、表示時にも改行が反映されます。

### セリフ内での構文記号の扱い

Lexer のモード切り替え（`「` で DIALOGUE モードに入る）により、セリフ内では通常の構文ルールが適用されません。以下はすべてプレーンテキストとして保持されます。

- `$$` → コメントにならない
- `おわり` → ブロック終了にならない
- `→` → ジャンプにならない
- その他の構文記号

```daihon
「$$ これはコメントではなく、そのまま表示される」
「おわりと言っても、ここではセリフの一部」
```

---

## 6.5. セリフ内の記号エスケープ

セリフ中で構文記号を文字としてそのまま表示したい場合は、同じ記号を 2 つ重ねて記述します。

| 記述 | 出力される文字 |
|------|----|
| `＜＜` | `＜` |
| `＞＞` | `＞` |
| `「「` | `「` |
| `」」` | `」` |

```daihon
「数学の問題：3＜＜5は正しい？」   $$ → 3＜5は正しい？
「彼は「「やだ」」と言った」       $$ → 彼は「やだ」と言った
```

> **実装上の注意:** ANTLR の Lexer はルール定義順で優先かつ最長一致で動作するため、二重化エスケープ用ルール（`＜＜`、`＞＞` 等）を通常の記号ルール（`＜`、`＞` 等）より前に定義してください。
