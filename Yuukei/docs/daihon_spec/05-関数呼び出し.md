# 5. 関数呼び出し（アクション）

> システム（Unity 側など）に特定のアクションを実行させる、またはパラメータを変更するには `＜ ＞`（全角山括弧）を使用します。

---

## 5.1. 基本構文

```
＜関数名 引数1 引数2 名前付き引数=値＞
```

### 位置引数

スペースで区切って順番に指定します。

```daihon
＜動作 チョコ渡す＞
＜ランダム 1 100＞
```

### 名前付き引数

`引数名=値` の形式で指定します。位置引数と混在可能です。

```daihon
＜設定変更 音量=50 明るさ=80＞
＜エフェクト 爆発 速度=2 色=「赤」＞
```

名前付き引数の値には、リテラル値、変数参照、および `（）` で囲んだ算術式を記述できます。スペースを含む算術式は必ず `（）` で囲んでください。

```daihon
＜ダメージ計算 攻撃力=（基礎攻撃力 + 10） 防御無視=はい＞
＜表示 x=（画面幅 / 2） y=（画面高さ / 2）＞
```

### スペースを含む値の渡し方

関数に渡す文字列にスペースが含まれる場合は、引数区切りと誤認されないよう `「」` で囲んでください。

```daihon
＜ユーザー名変更 「山田 太郎」＞
```

> **実装上の注意:** `＜` `＞`（全角）は「関数呼び出し開始・終了」専用の記号です。半角 `<` `>` は比較演算子として使用されるため、全角と半角は明確に異なるトークンです。

---

## 5.2. 出現可能な位置

関数呼び出し `＜＞` は以下のすべての位置に出現できます。

| 位置 | 例 |
|------|---|
| 独立した文（1つまたは連続） | `＜笑顔＞`、`＜不安＞＜w3＞＜笑顔＞` |
| セリフの中 | `「＜笑顔＞おはよう」` |
| 条件式の中（戻り値として） | `※（＜ランダム 1 100＞ 50 以下）「ラッキー！」` |
| 代入の右辺（戻り値として） | `天気=＜現在の天気＞` |
| 算術式の中（戻り値として） | `ダメージ=＜攻撃力計算 武器＞ * 2` |

独立した文としての関数呼び出しは、1 行に複数を連続して記述できます。

```daihon
＜不安＞＜w3＞＜笑顔＞
```

セリフ（`「」`）内の関数呼び出しは Lexer のモード切り替えで処理されます。`「` でセリフモードに入り、セリフモード内で `＜` を検出すると関数呼び出しモードに遷移します。詳細は [§6.4](06-セリフとブロック.md#64-セリフの内部構造) を参照してください。

---

## 5.3. セリフ内の変数展開

セリフの中で `＜変数名＞` と記述すると、その変数の値が文字列として展開・表示されます。この構文はセリフ内でのみ有効であり、代入文の右辺等では使用できません。

```daihon
「こんにちは＜ユーザー名＞さん、今日は＜月＞月＜日＞日ですね」
```

> **実装上の注意:** `＜＞` が関数呼び出しなのか変数展開なのかは、実行エンジンが識別子を関数名・変数名のどちらとして解決するかで判断します。パーサーはどちらも同じ構文構造として解析するだけに留めてください。

---

## 5.4. 戻り値を持つ関数

関数呼び出しはアクション実行だけでなく、戻り値を返すことができます。戻り値を持つ関数は、条件式・代入の右辺・算術式で使用できます。

```daihon
$$ 条件式内
※（＜所持アイテム りんご＞ = はい）「りんごがあるね！」
※（＜ランダム 1 100＞ 50 以下）「ラッキー！」

$$ 代入の右辺
天気=＜現在の天気＞
乱数=＜ランダム 1 100＞

$$ 算術式との組み合わせ
ダメージ=＜攻撃力計算 武器＞ * 2
```

関数が戻り値を返すかどうか・戻り値の型は、実行エンジン側の関数定義に委ねられます。パーサーは `＜＞` をすべて同一の構文構造として解析するだけに留めてください。戻り値を返さない関数を条件式や代入の右辺で使用した場合は実行時エラーとなります。
