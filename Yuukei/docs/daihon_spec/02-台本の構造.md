# 2. 台本の構造

> 台本は「イベント」を頂点とし、その中にオプションの「前提条件」「初期値」ブロック、および複数の「シーン」が含まれる階層構造を持ちます。

---

## 2.1. イベント名の宣言

```
## イベント名
```

その台本が何のイベントであるかを示すタイトルです。**1 つの台本ファイルにつき 1 つのイベント**を定義します。

---

## 2.2. 前提条件ブロック（オプション）

イベント宣言の直後に記述できる特別なブロックです。イベント全体の実行可否を判定するための条件を定義します。

- `前提条件:` で開始し、次のセクション宣言（`初期値:` または `###`）が出現すると暗黙的に閉じます。
- ブロック内で許可される構文は **条件付き `→おわり`** のみです。

```daihon
## バレンタインイベント

前提条件:
※（好感度 0 以下）→おわり
```

---

## 2.3. 初期値ブロック（オプション）

前提条件ブロックの後（前提条件がない場合はイベント宣言の直後）に記述できる特別なブロックです。変数のデフォルト値を設定します。

- `初期値:` で開始し、最初の `###`（シーン宣言）が出現すると暗黙的に閉じます。
- ブロック内で許可される構文は **変数の代入文**（1 行に 1 つ）のみです。

**実行エンジンの挙動:** セーブデータにその変数が存在しない（未定義の）場合のみ、このブロックの値を代入します。

> **評価順序に関する注意:**
> 台本上の記述順は `前提条件:` → `初期値:` ですが、システム内部ではイベント開始時にまず初期値をセットし、その後に前提条件を判定します。そのため、前提条件の条件式内で初期値ブロックの変数をエラーなく参照できます。

```daihon
初期値:
2月14日に起動しなかった=はい
朝にユーザーと会話した=いいえ
```

---

## 2.4. ブロックの順序と省略規則

イベント宣言直後のブロック配置順序は以下の通り固定です。

```
## イベント名
前提条件:    （省略可）
初期値:      （省略可）
### シーン名  （1つ以上必須）
```

- `前提条件:` と `初期値:` はどちらも省略可能。両方省略して `###` から始めることもできます。
- `前提条件:` と `初期値:` の順序を入れ替えることはできません。

---

## 2.5. シーンの宣言

```
### シーン名
```

イベント内で、特定のタイミングや条件ごとに処理を分けるための見出しです。**1 つのイベントに 1 つ以上のシーンが必要です。**

シーン名はジャンプ（[§7.3](07-制御構文.md#73-シーンへのジャンプ)）の宛先ラベルとして使用されます。実行のきっかけは `合図:` と `条件:`（§2.6 参照）で定義します。

---

## 2.6. シーンのトリガー（合図・条件）

各シーンには、実行のきっかけとなる「合図」と、追加の「条件」をオプションで指定できます。

### 2.6.1. `合図:` — システムイベントトリガー

シーンが実行される「きっかけ」となるシステムイベントを指定します。`＠`（全角アットマーク）を接頭辞とした識別子で表記します。

```daihon
### 起動時の挨拶
合図: ＠起動時
「おはよう！」
```

複数のシステムイベントを `または` で列挙することもできます。`かつ` は許可されません（2 つのイベントが同時に発生する場面は想定されないため）。

```daihon
### クリック反応
合図: ＠クリック または ＠ダブルクリック
「呼んだ？」
```

> **構文上の注意:** `合図:` の後は条件式ではなく、「システムイベント名を `または` で連結したリスト」として専用の構文規則で解析されます。

システムイベントの一覧は言語仕様の外部で管理されます。パーサーは `＠` に続く識別子をシステムイベント名として受理するだけに留め、存在チェックは行いません。

**イベント名に使用できる文字:** ひらがな、カタカナ、漢字、英字、数字、`_`、`.`（階層区切り用）

例: `＠起動時`、`＠クリック`、`＠OnUpdate.OnDownloadBegin`

### 2.6.2. `条件:` — 条件式による実行判定

シーンの実行可否を判定する条件式を指定します。条件式の構文は [§4](04-条件式.md) に準拠します。

**`条件:` の後は `※` なしで `（条件式）` のみを記述します。** `※` はシーン本体内の条件マーカーであり、`条件:` 行では不要です。

```daihon
### 深夜の独り言
条件:（23:00~）
「……眠い」
```

### 2.6.3. `合図:` と `条件:` の組み合わせ

`合図:` と `条件:` は併用できます。この場合、「システムイベントが発生したタイミングで条件式を評価し、真であればシーンを実行する」という意味になります。

```daihon
### バレンタインクリック
合図: ＠クリック
条件:（月=2 かつ 日=14 かつ 好感度 50 以上）
＜笑顔＞
「チョコあげるね！」
```

### 2.6.4. 省略時の挙動

| `合図:` | `条件:` | 挙動 |
|---------|---------|------|
| あり | あり | イベント発生時に条件を評価し、真なら実行 |
| あり | なし | イベント発生時に無条件で実行 |
| なし | あり | 条件を常時監視し、成立したら実行 |
| なし | なし | **デフォルトシーン**（§2.6.5 参照） |

### 2.6.5. デフォルトシーンと複数シーンの同時成立

**複数シーンの順次実行:**
イベント発生時、実行エンジンはすべてのシーンの条件を上から順に評価・実行します。あるシーンの実行で変数が変化した場合、後続シーンの条件判定にはその変更が反映されます（逐次評価方式）。

```daihon
### シーンA
条件:（好感度 50 以上）
好感度=0

### シーンB
条件:（好感度 50 以上）
「好感度が高い！」
```

上記の場合、シーン A が実行されて好感度が 0 になるため、シーン B の条件は偽となり実行されません。

**デフォルトシーン（合図も条件もないシーン）:**

- 合図・条件付きのシーンが **1 つでも該当した場合**、デフォルトシーンは **実行されません。**
- **どのシーンにも該当しなかった場合に限り**、デフォルトシーンの中から **ランダムに 1 つだけ** 選ばれて実行されます。
- デフォルトシーンが 1 つだけの場合は、そのシーンがそのまま実行されます。

```daihon
## 挨拶イベント

### 朝の挨拶
条件:（時 < 12）
「おはよう！」

### 夜の挨拶
条件:（時 >= 18）
「こんばんは！」

### 通常A
「今日もいい天気だね」

### 通常B
「元気にしてた？」
```

- 午前中 → 「朝の挨拶」が実行。デフォルトシーンは実行されない。
- 夕方以降 → 「夜の挨拶」が実行。デフォルトシーンは実行されない。
- 昼間（12:00〜17:59） → 条件付きシーンが該当しないため、「通常A」か「通常B」のいずれかがランダムに実行。

### 2.6.6. 記述順序

`合図:` と `条件:` はシーン宣言（`###`）の直後、シーン本体の前に記述します。順序は `合図:` → `条件:` で固定です。

```
### シーン名
合図:   （省略可）
条件:   （省略可）
処理本体...
```
