# 4. 条件式

> 特定の条件を満たしたときのみ処理を実行するには `※（条件）` を使用します。

条件式は必ずアクション（セリフ、ジャンプ、関数呼び出し、代入、ブロック開始のいずれか）を伴う必要があります。条件式のみでアクションがない行はパースエラーです。

条件式の中では、関数呼び出しの戻り値も使用できます（[§5.4](05-関数呼び出し.md#54-戻り値を持つ関数) 参照）。

**条件式 `※（）` 内の `=` は常に等価比較です。** 代入の `=` とは異なります（[§3](03-変数.md) 参照）。

---

## 4.1. 一致・不一致

### 等価比較（`=` / `==`）

```daihon
※（朝にユーザーと会話した=はい）「おはよう」     $$ 後置形
※（朝にユーザーと会話した == はい）「おはよう」   $$ 中置形
```

条件式内の `=` と `==` はどちらも等価比較として同じ意味です。`=` は日本語話者に直感的な表記、`==` はプログラミング経験者に馴染み深い表記として、好みで使い分けられます。

### 不一致比較（`!=`）

```daihon
※（朝にユーザーと会話した!=はい）「初めまして」
```

### 関数の戻り値との比較

```daihon
※（＜所持アイテム りんご＞ = はい）「りんごがあるね！」
```

---

## 4.2. 真偽値の省略形

真偽値変数の条件判定では、`=はい` を省略して変数名のみで記述できます。「その変数が `はい` である」という条件として評価されます。

```daihon
※（朝にユーザーと会話した）「またね」
$$ ↑ 以下と同義 ↓
※（朝にユーザーと会話した=はい）「またね」
```

---

## 4.3. 大小比較

### 後置記法（日本語形）

日本語の語順に合わせ、比較演算子を値の後ろに記述します。**後置形の左辺は変数名のみに制限されます（算術式は不可）。**

| 構文 | 意味 |
|------|------|
| `※（変数名 値 未満）` | `<` |
| `※（変数名 値 以下）` | `<=` |
| `※（変数名 値 以上）` | `>=` |
| `※（変数名 値 超える）` | `>` |

数値リテラルと比較演算子の間のスペースは省略可能です（例: `10未満` と `10 未満` は同じ）。比較演算子が予約語であり、Lexer が数値直後でも別トークンとして認識するためです。

> **実装上の注意（パーサー指針）:**
> パーサーの文法定義では、後置記法の左辺に算術式も**構文的には受理**してください。これにより `※（好感度 + 10 50以上）` のような入力でもパーサーがクラッシュせず、意味解析フェーズで「後置記法の左辺に算術式は使えません。中置記法（`>=` 等）を使ってください」という親切なエラーメッセージを返せます。

### 中置記法（記号形）

算術式を含む比較や、記号形式を好む場合に使用します。**中置形では左辺に算術式を使用可能です。**

| 演算子 | 意味 |
|--------|------|
| `<` | 未満 |
| `<=` | 以下 |
| `>=` | 以上 |
| `>` | 超える |
| `==` | 等しい |
| `!=` | 等しくない |

```daihon
※（好感度 + 10 >= 50）「好き！」
※（ダメージ * 2 > 防御力）「効いた！」
※（好感度 >= 50）「好き！」    $$ 算術式なしでも中置形は使用可能
```

後置形と中置形はどちらも常に使用可能です。算術式を左辺に含む場合のみ中置形が必須です。

---

## 4.4. 範囲比較（`~` 演算子）

数値の範囲を簡潔に指定できます。**範囲は両端を含む（閉区間）**として評価されます。

```daihon
※（好感度 10~50）「普通だね」    $$ ≡ 好感度 10 以上 かつ 好感度 50 以下
※（好感度 ~50）「まだまだ」      $$ ≡ 好感度 50 以下
※（好感度 10~）「いい感じ」      $$ ≡ 好感度 10 以上
```

---

## 4.5. 組み込み変数（時間）

システム側で常に提供される変数です。通常の変数と同様に条件式や算術式で使用できます。

| 変数名 | 説明 | 型 |
|--------|------|-----|
| `年` | 現在の年 | 数値 |
| `月` | 現在の月 | 数値 |
| `日` | 現在の日 | 数値 |
| `曜日` | 現在の曜日（`「月」`〜`「日」`） | 文字列 |
| `週` | その月の第何週か | 数値 |
| `時` | 現在の時（0〜23） | 数値 |
| `分` | 現在の分（0〜59） | 数値 |
| `秒` | 現在の秒（0〜59） | 数値 |
| `ミリ秒` | 現在のミリ秒（0〜999） | 数値 |

```daihon
※（月=2 かつ 日=14）「バレンタインだ！」
※（曜日=「水」）「週の真ん中だね」
※（時 >= 22）「もう遅いよ」
```

---

## 4.6. 時間の範囲指定（専用構文）

日常的な時間指定を簡略化するために、`hh:mm` 形式を専用トークンとして扱います。時間範囲は内部的に真偽値を返す式として評価され、他の条件と `かつ` / `または` で組み合わせ可能です。

組み込み変数 `時`・`分` でも同等の条件を記述できますが、時間範囲構文の方が簡潔で可読性が高いため、こちらの使用を推奨します。

```daihon
※（15:00~18:30）「おやつの時間だ」         $$ 範囲
※（~8:00）「おはよう」                     $$ 〜まで
※（18:30~）「こんばんは」                  $$ 〜以降
※（15:00~18:30 かつ 好感度 10 以上）「一緒におやつ食べよう！」  $$ 複合
```

> **実装上の注意:** `TIME` トークンは `数字+ : 数字+` の形式で、`:` の前に数字が必要です。セクション宣言の `:` は直前が日本語文字（`前提条件`、`初期値` 等）であるため、`TIME` トークンと衝突しません。

---

## 4.7. 複合条件（かつ・または）

複数の条件を組み合わせる場合、基本的に左から右へ順に評価されます。評価の優先順位を変更するには `（ ）` / `( )` で囲んで明示します。

```daihon
$$ AND
※（月=2 かつ 日=14）「バレンタインだ！」

$$ OR
※（曜日=「土」 または 曜日=「日」）「休みだね」

$$ 丸括弧による優先順位の変更
※（好感度 50 以上 かつ （天気=「晴れ」 または 曜日=「日」））「お出かけしよう！」
$$ → まず「天気が晴れ、または日曜日」を評価し、その結果と「好感度50以上」を AND で評価
```
