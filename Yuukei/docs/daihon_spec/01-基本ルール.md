# 1. 基本ルール（字句解析の前提条件）

> この章では、台本ファイルの読み込み時に適用される文字の正規化やトークン分割など、字句解析（Lexer）レベルの前提条件を定義します。

---

## 1.1. 文字の正規化（全角・半角の同一視）

入力されたすべての全角英数字および一部の全角記号は、対応する半角文字に対応付けされます。この対応付けは前処理ではなく、ANTLR の Lexer ルールによって全角・半角の両方を許容する形で実現します。

### 正規化対象の記号

| 全角 | 半角 | 用途 |
|------|------|------|
| `＝` | `=` | 代入・比較 |
| `＃` | `#` | 見出し宣言 |
| `：` | `:` | ブロック開始 |
| `＄` | `$` | コメント |
| `＿` | `_` | 一時変数接頭辞 |
| `！` | `!` | 不一致演算子 |
| `～` | `~` | 範囲演算子 |
| `＠` | `@` | システムイベント接頭辞 |
| `．` | `.` | 小数点 |
| 全角英数字 | 半角英数字 | 変数名・数値 |
| 全角スペース | 半角スペース | 空白の統一 |

### 正規化しない記号

| 記号 | 理由 |
|------|------|
| `＜` `＞`（全角） | 関数呼び出し専用。半角 `<` `>` は比較演算子として別用途 |
| `（` `）`（全角） | Lexer で半角 `(` `)` と同一トークンとして扱う（正規化ではなく Lexer 側で吸収） |
| `「` `」` | セリフ記号。正規化対象外 |
| `→` | 制御構文専用 |
| `※` | 条件式専用 |

### セリフ内テキストの保護

`「」`（カギカッコ）内のテキストは正規化の対象外です。セリフはユーザーに表示される内容であり、作者が記述した全角・半角をそのまま保持します。

ただし、セリフ内の構文記号（`＜` `＞` `「` `」`）は正規化対象外であっても、関数呼び出しやエスケープ（[§6.5](06-セリフとブロック.md#65-セリフ内の記号エスケープ) 参照）の構文解析は通常通り行われます。

### エスケープ記号 `「「` `」」` への配慮

正規化処理で `「」` の内外を追跡する際、`「「`（エスケープされたカギカッコ）はセリフの開始・終了とみなさず、2文字をそのまま保持してください。

---

## 1.2. スペースとインデント

スペース（全角・半角）はトークンの区切りとして機能します。連続する複数のスペースは1つと同等に扱われます。行頭のスペース（インデント）は見た目を整えるためだけのものであり、構文上の意味を持ちません。

ただし、`「」` 内のスペースはセリフの一部としてそのまま保持されます。

### スペースとトークン分割の関係

構文記号（`=`、`＜`、`＞`、`「`、`」`、`※`、`→` 等）の前後ではスペースがなくてもトークンが分割されます。一方、**識別子と数値リテラルの間にはスペースが必要**です。スペースがない場合、Lexer は最長一致により1つのトークンとして認識します。

```daihon
好感度=10         $$ → 好感度 = 10（= は構文記号なのでスペース不要）
好感度 10 未満    $$ → 好感度 / 10 / 未満（識別子と数値の間にスペース必要）
好感度 10未満     $$ → 好感度 / 10 / 未満（未満 は予約語なので数値直後でも分割）
好感度10未満      $$ → 好感度10未満（1つの IDENTIFIER として誤認識）
```

---

## 1.3. 改行

改行は文の区切りとして機能します。各文は原則 1 行に 1 つ記述します（[§6.1](06-セリフとブロック.md#61-文ステートメントの種類と1行1文ルール) 参照）。空行は無視されます。

セリフ（`「」`）内の改行のみ、セリフの一部としてそのまま保持されます（[§6.4](06-セリフとブロック.md#64-セリフの内部構造) 参照）。

---

## 1.4. コメント

台本の中に、実行されないメモを残すことができます。`$$` から行末までが一行コメントとして扱われます。

**セリフ内（`「」` 内）の `$$` はコメントになりません。** Lexer のモード切り替え（[§6.4](06-セリフとブロック.md#64-セリフの内部構造) 参照）により、セリフ内ではコメントルールが適用されず、`$$` はプレーンテキストとして保持されます。

```daihon
$$ ここはコメント。実行されない
好感度=10 $$ここにメモを書くこともできる

「$$ これはコメントではなく、セリフの一部」
```

---

## 1.5. ファイルエンコーディング

台本ファイルは **UTF-8（BOM なし）** で記述してください。
