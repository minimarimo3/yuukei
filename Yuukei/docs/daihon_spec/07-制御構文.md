# 7. 制御構文（終了とジャンプ）

> イベントの進行を制御するために `→`（矢印）を使用します。処理の強制終了や、同じイベント内の別シーンへのジャンプ（移動）が可能です。

`→` はブロック内でもトップレベル（シーン直下）でも使用できます。

ジャンプは **goto 方式**です。現在の実行位置を破棄し、指定先に無条件で移動します。呼び出し元に戻る機能はありません。

**無限ループ防止:** 実行エンジンはジャンプ回数を追跡し、1 イベントあたり **1000 回**を超えた場合、実行時エラーとして強制終了します。

---

## 7.1. イベントの終了（`→おわり`）

イベント全体の処理を完全に終了させます。複数シーンの順次実行中（[§2.6.5](02-台本の構造.md#265-デフォルトシーンと複数シーンの同時成立) 参照）に実行された場合、残りのシーンはすべてスキップされ、イベントが即座に終了します。

```
→おわり
```

```daihon
※（好感度 0 以下）→おわり
```

---

## 7.2. シーンの終了（`→シーンおわり`）

現在実行中のシーンのみを終了し、順次実行リスト（[§2.6.5](02-台本の構造.md#265-デフォルトシーンと複数シーンの同時成立) 参照）の次のシーンの条件評価に進みます。深いネストの中からシーンを一気に抜けたい場合に有用です。

```
→シーンおわり
```

```daihon
※（特別フラグ）:
    ※（時 < 12）:
        「おはよう！特別な朝だね」
        →シーンおわり  $$ シーンを抜け、次のシーン評価に進む
    おわり
    「特別な日だね」
おわり
「通常の処理」
```

### `→おわり` と `→シーンおわり` の違い

| 構文 | 効果 |
|------|------|
| `→おわり` | イベント全体を終了。残りのシーンは一切実行されない |
| `→シーンおわり` | 現在のシーンだけを終了。次のシーンの評価に進む |

---

## 7.3. シーンへのジャンプ

同じイベント内の別の `### シーン名` に処理を移動させます。条件を満たすまで同じ質問を繰り返すループ処理（選択待ちなど）に活用できます。

**ジャンプを実行すると、複数シーンの順次実行リストは破棄されます。** ジャンプ先のシーン実行が完了しても、元の順次実行リストには戻りません。

```
→ジャンプ先のシーン名
```

```daihon
## ヤンデレ質問イベント

前提条件:
※（好感度 50 未満）→おわり

### 最初の質問
「私と仕事、どっちが大事なの？」
→選択待ちループ

### 選択待ちループ
＜選択肢表示 私 仕事＞
※（選択結果=「仕事」）:
    「聞き間違いだよね？もう一回聞くね？」
    →選択待ちループ
※それ以外:
    「ずっと一緒だよ……」
    →おわり
おわり
```

### ジャンプと順次実行の関係

```daihon
### シーンA
条件:（フラグA）
→シーンC  $$ シーンC に飛ぶ。順次実行リストは破棄

### シーンB
条件:（フラグB）
「シーンBです」  $$ シーンA でジャンプした場合、実行されない

### シーンC
「シーンCです」
```

シーン A とシーン B の条件が両方成立した場合の実行順:

1. シーン A が実行される
2. `→シーンC` でシーン C にジャンプ（順次実行リスト破棄）
3. シーン C が実行される
4. **シーン B は実行されない**

### 3 種類の制御構文まとめ

| 構文 | 効果 | 順次実行リスト |
|------|------|-------------|
| `→おわり` | イベント全体を終了 | 破棄（イベント終了） |
| `→シーンおわり` | 現在のシーンを終了し、次のシーンの評価に進む | 維持 |
| `→シーン名` | 指定シーンに無条件で移動 | 破棄 |

> **実装上の注意:** パーサーは `→` の後に続く文字列を「ジャンプ先の識別子」または予約語（`おわり`、`シーンおわり`）として解析するだけに留めてください。シーン名の存在チェックは実行エンジン側の責務です。

# 8. エラーリカバリ

パーサーが構文エラーを検出した場合、**そのイベントファイル全体の読み込みを中止します。** 実行エンジンは構文エラーのあるファイルを無視し、他のイベントファイルの処理を続行します。

エラーメッセージには、エラー発生行の行番号と、期待されたトークンの情報を含めてください。
