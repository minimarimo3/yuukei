# 3. 変数（フラグ）の操作と代入

> キャラクターの記憶や状態（フラグ）を操作するには `=` を使って値を代入します。値はすべて自動保存されます（一時変数を除く。§3.4 参照）。

---

## 基本構文

```
変数名=値
```

**代入文は 1 行に 1 つだけ記述できます。** 同じ行に複数の代入文や、代入文と他のアクション（セリフ・関数呼び出し等）を混在させることはできません。

```daihon
朝にユーザーと会話した=はい
好感度=10
```

代入の右辺には、リテラル値のほか、変数参照、算術式（§3.6 参照）、関数呼び出しの戻り値（[§5.4](05-関数呼び出し.md#54-戻り値を持つ関数) 参照）も記述できます。

```daihon
好感度=好感度 + 10
表示名=ユーザー名
天気=＜現在の天気＞
乱数=＜ランダム 1 100＞
```

### `=` の意味の使い分け

| コンテキスト | `=` の意味 |
|---|---|
| 条件式 `※（）` の**外側** | **代入** |
| 条件式 `※（）` の**内側** | **等価比較** |

条件式内で代入を行うことはできません。代入は必ず条件式の外側で記述してください。

---

## 3.1. 変数名の規則

**使用できる文字:** ひらがな、カタカナ、漢字、英字（全角・半角）、数字（全角・半角）、`_`（全角・半角）

**制約:**
- 数字始まりの変数名は不可。数字始まりを許可すると `10未満` などで数値リテラルと比較演算子の分割ができなくなるためです。
- 先頭に `_` をつけると一時変数になります（§3.4 参照）。

**予約語との関係:**
予約語（§3.2 参照）そのものは変数名に使えませんが、**予約語を部分文字列として含む変数名は許可されます。** ANTLR4 の Lexer は最長一致で動作するため、たとえば `未満足度` は `IDENTIFIER` トークン 1 つとして正しく認識されます。

| 変数名 | 可否 | 理由 |
|---|---|---|
| `未満足度` | ✅ | IDENTIFIER として認識 |
| `未満` | ❌ | 予約語 |
| `おわりに` | ✅ | IDENTIFIER として認識 |
| `おわり` | ❌ | 予約語 |

**使用できない文字（構文記号）:**
`=` `!` `＜` `＞` `<` `>` `(` `)` `（` `）` `「` `」` `$` `:` `→` `※` `#` `~` `＠` `+` `-` `*` `/` `%` `.` およびスペース（全角・半角）

---

## 3.2. 予約語

以下の語は変数名として使用できません。ただし、部分文字列として含む変数名は許可されます（§3.1 参照）。

| 予約語 | 用途 |
|--------|------|
| `おわり` | ブロック終了 / イベント終了 |
| `シーンおわり` | シーン終了 |
| `はい` / `いいえ` | 真偽値リテラル |
| `未満` `以下` `以上` `超える` | 比較演算子（後置形） |
| `かつ` `または` | 論理演算子 |
| `前提条件` `初期値` | セクション宣言 |
| `合図` `条件` | シーントリガー宣言 |
| `あるいは` | Else If 構文 |
| `なら` | 条件式の接尾辞（省略可） |
| `それ以外` | Else 構文 |

---

## 3.3. 変数の型と値の表現

台本で扱うデータには **真偽値・数値・文字列** の 3 種類の型があります。

### 真偽値（フラグ）

予約語 `はい` または `いいえ` と記述します。囲み記号は不要です。

```daihon
朝にユーザーと会話した=はい
```

### 数値

数字をそのまま記述します。囲み記号は不要です。

| 種類 | 例 |
|------|---|
| 整数 | `10`, `42`, `１０`, `54３21` |
| 小数 | `3.14`, `0.5`, `３．１４` |
| 負数 | `-10`, `-１０`（単項マイナス。AST では `UnaryMinus(10)` と表現） |
| 16進数 | `0xFF`（`0x` プレフィックス、半角強制） |
| 8進数 | `0o77`（`0o` プレフィックス、半角強制） |
| 2進数 | `0b1010`（`0b` プレフィックス、半角強制） |

### 文字列

必ず `「」`（カギカッコ）で囲んで記述します。

```daihon
誕生日=「2月28日」
```

---

## 3.4. 変数の永続化（セーブ）と一時変数

**永続変数（自動セーブ）:**
台本内で代入した変数は、すべてシステムによって自動保存され、次回起動時にも引き継がれます。

**一時変数（保存されない変数）:**
変数名の先頭に `_`（全角・半角問わず）をつけると一時変数として扱われ、アプリ終了時またはイベント終了時に破棄されます。選択肢ループや一時的なカウンタなどに活用してください。

```daihon
_ループ回数=1
```

---

## 3.5. 型に関するルール

### 型の不一致による比較

異なる型同士の比較（例: 数値と文字列）は常に **偽** と評価されます。エラーにはなりません。

```daihon
好感度=10
※（好感度=「10」）「同じだね」  $$ 数値と文字列の比較 → 常に偽
```

### 型の再代入（禁止）

一度値が代入された変数に対して、異なる型の値を再代入することはできません。実行時エラーになります。

```daihon
好感度=10
好感度=はい      $$ 数値 → 真偽値 → エラー
好感度=「abc」 + 好感度  $$ 右辺は「abc10」（文字列）→ 数値変数への文字列代入 → エラー
```

### 未定義変数の参照（エラー）

初期値ブロックや事前の代入で定義されていない変数を参照すると、実行時エラーになります。

> **実装上の注意:** 型再代入の禁止・未定義変数の参照エラーは実行時（意味解析）の責務です。パーサーは構文的に受理し、AST に記録するだけに留めてください。

---

## 3.6. 算術演算

代入の右辺や条件式の中で算術演算を使用できます。

### 演算子一覧（すべて半角）

| 演算子 | 意味 | 優先順位 |
|--------|------|---------|
| `+` | 加算 / 文字列結合（単項プラス） | 低（単項は高） |
| `-` | 減算（単項マイナス） | 低（単項は高） |
| `*` | 乗算 | 高 |
| `/` | 除算 | 高 |
| `%` | 剰余 | 高 |

優先順位は通常の数学と同じです。丸括弧 `（）` / `()` で優先順位を変更できます。

```daihon
好感度=好感度 + 10
ダメージ=(攻撃力 - 防御力) * 2
残り=合計 % 3
```

**整数除算のルール:** 整数同士の除算は整数結果（切り捨て）を返します。小数を含む場合は小数結果を返します。

```daihon
結果=10 / 3     $$ → 3（整数÷整数 → 切り捨て）
結果=10.0 / 3   $$ → 3.333...（小数あり → 小数結果）
```

### 3.6.1. 文字列結合

`+` 演算子は、オペランドの型に応じて加算または文字列結合として動作します。

| 左辺 | 右辺 | 動作 |
|------|------|------|
| 数値 | 数値 | 数値の加算 |
| 文字列 | 文字列 | 文字列の結合 |
| 文字列 | 数値 / 真偽値 | 自動変換して結合 |
| 数値 / 真偽値 | 文字列 | 自動変換して結合 |

**片方でも文字列であれば、もう片方を自動変換して結合します。** 変換ルール:

- 数値 → 数字文字列（例: `50` → `「50」`）
- 真偽値 → `「はい」` / `「いいえ」`

結合結果は常に文字列型です。§3.5 の型再代入禁止ルールにより、数値型変数への文字列結合結果の代入は実行時エラーとなります。

```daihon
メッセージ=「好感度は」 + 好感度 + 「です」   $$ → 「好感度は50です」
ラベル=「レベル」 + 10                         $$ → 「レベル10」
状態テキスト=「フラグ:」 + フラグ変数           $$ → 「フラグ:はい」（フラグ変数=はい の場合）
```

> **実装上の注意:** `+` が加算・文字列結合のどちらになるかはオペランドの型に基づき実行時に決定されます。パーサーは `+` をすべて同一の二項演算子として AST に記録するだけに留めてください。
