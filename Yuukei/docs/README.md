# 有景 / Yuukei

**デバイスの中に生きる、3Dデスクトップマスコット**

> チャットの相手ではなく、あなたのPCに住んでいる存在。

---

## これは何か

Yuukei（有景）は Unity + VRM で動作する OSS のデスクトップマスコットです。

既存のデスクトップマスコットとは一線を画す、**「デバイスへの居住」**という設計思想のもとに作られています。キャラクターはあなたが作業しているディスプレイに表示されている内容を把握し、時刻や状況に応じて自然に反応します。ときには画面から姿を消し、フォルダの奥深くでひとり時間を過ごすこともあるでしょう。

会話相手として使うのではなく、**PCとともに生活する存在**として設計されています。

---

## 背景とコンセプト

「カゲロウプロジェクト（メカクシティアクターズ）」に登場するキャラクター「エネ」に着想を得ています。

エネは電子デバイスの内側に住んでいるような存在で、PCのフォルダを閲覧してその内容について話しかけてきたり、機嫌が悪くなるとスマートフォンに移動してYouTubeを眺めたりします。彼女のような——**デバイスの中に実在する感覚**——を現代の技術で再現することが Yuukei の出発点です。

Yuukei はキャラクターの「使い道」ではなく「存在感」そのものに価値を置きます。

理念・対象ユーザー・機能方針の整理は [`yuukei_spec/00-プロダクトコンセプト.md`](yuukei_spec/00-プロダクトコンセプト.md) にまとめています。

## 主な特徴

### 3D キャラクター（VRM）

Unity と VRM フォーマットを採用することで、イラストや Live2D とは次元の異なる表現力と実在感を実現しています。キャラクターは口パク・モーション再生・浮遊アニメーションを持ちます。

### 台本（Daihon）DSL

キャラクターの振る舞いは `.daihon` という独自のスクリプトファイルで定義します。日本語ネイティブな記法で条件分岐・変数操作・関数呼び出しを記述でき、コーディング経験がなくても読み書きできるよう設計されています。

```daihon
## はじめての挨拶イベント
初期値：
    名前を教えてもらった=いいえ
    ユーザー名=「ほげほげくん」

### 最初のあいさつ
合図: ＠起動時 または　＠あれ

「＜笑顔＞やっほー！起動してくれてありがとう！」
「今日は何して遊ぶ？」

$$ ユーザーの名前を初期設定する（まだ決まっていない場合）
※（ユーザー名 = 「」）:
    「そういえば、まだ名前を聞いてなかったね。なんて呼べばいい？」
    ＜名前入力ダイアログ＞
    「＜ユーザー名＞さん……いい名前だね！」
    名前を教えてもらった=はい
おわり

※（名前を教えてもらった）:
    「改めて、よろしくね！＜ユーザー名＞さん！」
おわり
```

詳細は [`daihon_spec/`](daihon_spec/index.md) を参照してください。

### トリガーシステム

キャラクターは外部イベントを起点に行動します。組み込みトリガーとして以下が用意されています。

| トリガー | 概要 |
|---------|------|
| 起動時 | アプリ起動直後に一度だけ発火 |
| クリック | キャラクターをクリックしたとき |
| 時刻 | 指定した時刻・曜日に発火 |
| システムイベント | CPU高負荷やバッテリー低下を検知 |
| エクスプローラー項目 | 指定ファイルの上にキャラが重なったとき |
| ファイルドロップ | ウィンドウにファイルをドロップしたとき |
| 定期確認 | 一定間隔で状態条件を評価する |

### LLM 連携

クラウド（OpenAI 互換 API）またはローカル（Ollama）の LLM と連携し、台本内から `LLM` 関数として呼び出せます。会話履歴は永続化されます。API キーは OS の認証情報ストアに安全に保管され、設定ファイルには書き込まれません。

### プラグイン・パッケージシステム

C# DLL プラグインでカスタムトリガーや台本関数を追加できます。テーマ・スクリプト・プラグインを ZIP にまとめた「パッケージ」としてドロップ一発でインストール可能です。

## アーキテクチャ

Pure DI を主軸とした 4 層アーキテクチャを採用しています。

```
UI 層        ← SettingsUIManager / SpeechBubble / FileDropRouter
アプリ層     ← CharacterManager / TriggerManager / LLMBridge / PackageManager
ドメイン層   ← DaihonRunner / FloatingMotionController / VRMLipSync / Triggers
インフラ層   ← Platform 抽象化 / PluginLoader / データ永続化
```

`PlatformBootstrapper` が唯一の Composition Root として全コンポーネントを組み立てます。`static Instance` パターンは使用せず、依存はコンストラクタ・`[SerializeField]`・`Initialize()` メソッドで明示します。

詳細なアーキテクチャ設計は [`yuukei_spec/`](yuukei_spec/index.md) を参照してください。
