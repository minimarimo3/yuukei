# 11 プラットフォーム抽象化

## 1. 概要

OS 依存の処理をインターフェースで抽象化します。  
`PlatformServiceFactory` がコンパイル時のプリプロセッサシンボルに基づいて  
適切な実装を選択します。

## 2. IAppIntegration

OS レベルのウィンドウ・システム統合機能を提供するインターフェースです。

```csharp
public interface IAppIntegration : IDisposable
{
    /// <summary>
    /// システムトレイアイコンを初期化する。
    /// </summary>
    /// <param name="onSettingsRequested">「設定」メニュー選択時のコールバック。</param>
    /// <param name="onQuitRequested">「終了」メニュー選択時のコールバック。</param>
    void InitializeTray(Action onSettingsRequested, Action onQuitRequested);

    /// <summary>
    /// アプリウィンドウをマルチモニター全体にリサイズする。
    /// </summary>
    void SetupMultiMonitor();

    /// <summary>
    /// アプリウィンドウをタスクバーと Alt+Tab から非表示にする。
    /// </summary>
    void HideFromTaskbar();

    /// <summary>
    /// メインスレッドのコールバックキューを処理する（毎フレーム Update から呼ぶ）。
    /// </summary>
    void Tick();
}
```

## 3. ICredentialStorage

認証情報（API キー等）を安全に保存するインターフェースです。

```csharp
public interface ICredentialStorage
{
    /// <summary>指定キーで認証情報を保存する。</summary>
    void Save(string key, string secret);

    /// <summary>指定キーの認証情報を読み込む。存在しない場合は null を返す。</summary>
    string Load(string key);

    /// <summary>指定キーの認証情報を削除する。</summary>
    void Delete(string key);
}
```

## 4. PlatformServiceFactory

```csharp
public static class PlatformServiceFactory
{
    public static IAppIntegration CreateAppIntegration()
    {
#if UNITY_STANDALONE_WIN
        return new WindowsAppIntegration();
#else
        return new DummyAppIntegration();
#endif
    }

    public static ICredentialStorage CreateCredentialStorage()
    {
#if UNITY_STANDALONE_WIN
        return new WindowsCredentialStorage();
#else
        return new DummyCredentialStorage();
#endif
    }
}
```

## 5. Windows 実装

### 5.1 WindowsAppIntegration

| 機能 | 実装 |
|------|------|
| システムトレイ | `System.Windows.Forms.NotifyIcon`（専用 STA スレッドで動作） |
| タスクバー非表示 | `User32.dll` の `SetWindowLong` で `WS_EX_APPWINDOW` 除去 + `WS_EX_TOOLWINDOW` 追加 |
| マルチモニター | `User32.dll` の `GetSystemMetrics` で仮想スクリーン全体のサイズを取得してウィンドウをリサイズ |
| メインスレッド橋渡し | `ConcurrentQueue<Action>` で STA スレッドから Unity メインスレッドへコールバックを渡す |

**STA スレッドの必要性**  
`System.Windows.Forms.NotifyIcon` は STA（Single-Threaded Apartment）スレッドで実行する必要があります。  
専用スレッドを作成し、`Application.Run()` でメッセージループを回します。

**Win32 API 定数**

| 定数 | 値 | 用途 |
|------|----|------|
| `GWL_EXSTYLE` | `-20` | 拡張ウィンドウスタイル取得 |
| `WS_EX_APPWINDOW` | `0x00040000` | タスクバー表示フラグ |
| `WS_EX_TOOLWINDOW` | `0x00000080` | ツールウィンドウフラグ |

### 5.2 WindowsCredentialStorage

```
Save(key, secret)
  → advapi32.dll CredWriteW
    CREDENTIAL.Type    = CRED_TYPE_GENERIC
    CREDENTIAL.Target  = "Yuukei_{key}"

```

## 6. フォールバック実装

### 6.1 DummyAppIntegration

非 Windows プラットフォーム用のノーオペレーション実装です。

```csharp
public class DummyAppIntegration : IAppIntegration
{
    public void InitializeTray(Action onSettingsRequested, Action onQuitRequested)
        => Debug.LogWarning("[DummyAppIntegration] システムトレイは非対応プラットフォームです。");

    public void SetupMultiMonitor() { }
    public void HideFromTaskbar()   { }
    public void Tick()              { }
    public void Dispose()           { }
}
```

### 6.2 DummyCredentialStorage

認証情報をメモリに保持するフォールバック実装です（再起動で失われる）。

```csharp
public class DummyCredentialStorage : ICredentialStorage
{
    private readonly Dictionary<string, string> _store = new();

    public void   Save(string key, string secret) => _store[key] = secret;
    public string Load(string key) => _store.TryGetValue(key, out var v) ? v : null;
    public void   Delete(string key) => _store.Remove(key);
}
```

起動時にログで警告を出します：  
`"[DummyCredentialStorage] 認証情報はメモリのみで管理されます（再起動後に失われます）。"`

## 7. ExplorerItemScanner（Windows 専用補助ツール）

### 7.1 概要

Windows エクスプローラー内の特定ファイルの画面上座標を取得する外部プロセスです。  
`Assets/StreamingAssets/ExplorerItemScanner/ExplorerItemScanner.exe` に配置します。

### 7.2 プロセス通信

```
起動引数:  ExplorerItemScanner.exe <targetFileName>
stdout:    SUCCESS:{left},{top},{width},{height}  （座標が見つかった場合）
           （見つからない場合は出力なし）
```

### 7.3 Unity 側の実装（ExplorerItemScanner.cs）

```csharp
public class ExplorerItemScanner : MonoBehaviour
{
    // 依存（Inspector から注入）
    [SerializeField] private Transform targetObject; // 対象 GameObject の Transform

    /// <summary>スキャンを開始する。</summary>
    public void StartScanning(string targetFileName);

    /// <summary>スキャンを停止する。</summary>
    public void StopScanning();
}
```

**stdout の解析**

```
"SUCCESS:100,200,50,30"
  → left=100, top=200, width=50, height=30
  → targetObject.position を仮想スクリーン座標からワールド座標に変換して更新
```

**デバウンス**：300 ms 以内の連続更新は無視します。

### 7.4 ExplorerItemScanner.exe（.NET スタンドアロン）

| 項目 | 内容 |
|------|------|
| 技術 | System.Windows.Automation（UIAutomation） |
| 監視対象 | `CabinetWClass`（エクスプローラーウィンドウ） |
| 検出方法 | List/DataGrid 要素内の指定ファイル名を検索 |
| イベント | Structure Changed + ウィンドウ移動イベントを購読 |
| デバウンス | 300 ms |
| ビルド | `cd ExplorerItemScanner && dotnet publish -c Release` |
