# 01 アーキテクチャ

## 1. 全体レイヤー構成

```
┌─────────────────────────────────────────────────────┐
│  UI 層                                               │
│  SettingsUIManager / PopupManager / SpeechBubble    │
├─────────────────────────────────────────────────────┤
│  アプリケーション層                                   │
│  PlatformBootstrapper（Composition Root）            │
│  CharacterManager / DaihonScenarioManager           │
│  TriggerManager / PackageManager / ThemeManager     │
├─────────────────────────────────────────────────────┤
│  ドメイン層                                          │
│  FloatingMotionController / VRMLipSync / VrmaPlayer │
│  DaihonRunner / YuukeiActionHandler                 │
│  各種 Trigger（TimeTrigger, ClickTrigger, ...）       │
├─────────────────────────────────────────────────────┤
│  インフラ層                                          │
│  Platform/（IAppIntegration, ICredentialStorage）   │
│  LLMBridge / FileDropRouter                         │
│  PluginLoader / PackageManager（I/O 部分）           │
└─────────────────────────────────────────────────────┘
```

## 2. 依存方向の規則

- **上位層は下位層にのみ依存** します（UI → App → Domain → Infra）。
- **Domain 層はインフラ・UI に依存してはなりません**。
- インターフェースを介した逆依存（DIP）を使用して循環依存を防ぎます。
- MonoBehaviour 間の依存は **SerializeField で Inspector から注入** するか、コンストラクタ相当の `Initialize(deps)` メソッドで渡します。

## 3. Pure DI 方針

### 3.1 基本ルール

```csharp
// ✅ 良い例：シグネチャから依存が明確
public class YuukeiActionHandler : MonoBehaviour, IUniTaskActionHandler
{
    [SerializeField] private CharacterManager characterManager;
    [SerializeField] private VRMLipSync lipSync;
    [SerializeField] private VrmaPlayer vrmaPlayer;
    [SerializeField] private FloatingMotionController floatingMotion;
    [SerializeField] private SpeechBubbleController speechBubble;
    [SerializeField] private LLMBridge llmBridge;
}

// ❌ 悪い例：隠れた依存（FindObjectOfType / static アクセス乱用）
public class YuukeiActionHandler : MonoBehaviour
{
    private void SomeMethod()
    {
        var mgr = FindObjectOfType<CharacterManager>(); // 隠れた依存
        CharacterManager.Instance.DoSomething();         // 同上
    }
}
```

### 3.2 PlatformBootstrapper（Composition Root）

`PlatformBootstrapper` は **唯一の Composition Root** です。  
ここでのみ `PlatformServiceFactory` を呼び出してインフラインスタンスを生成し、  
各コンポーネントへ注入（Inspector 参照 + Initialize 呼び出し）します。

```
PlatformBootstrapper
  ├── creates: IAppIntegration（via PlatformServiceFactory）
  ├── creates: ICredentialStorage（via PlatformServiceFactory）
  ├── wires: CharacterManager → (has SerializeField refs)
  ├── wires: YuukeiActionHandler → (has SerializeField refs)
  └── orchestrates: Boot sequence
```

### 3.3 Service Locator の使用制限

以下 **1 つのサービスのみ** Service Locator パターンを使用します。  
それ以外での `Instance` / `FindObjectOfType` 使用は禁止です。

| サービス | 理由 |
|---------|------|
| `IAppContext`（後述） | DSL 実行中など呼び出しスタックを経由して渡せない箇所でのみ使用 |

## 4. IAppContext（軽量 Service Locator）

アプリ全体で共有される少数のサービスを束ねる読み取り専用のコンテキストです。

```csharp
/// <summary>
/// アプリ全体で共有される基盤サービスへのアクセスポイント。
/// Service Locator として使用するが、登録できるサービスはここに列挙されたものだけに限定する。
/// </summary>
public interface IAppContext
{
    IAppIntegration    AppIntegration    { get; }
    ICredentialStorage CredentialStorage { get; }
    ConfigManager      ConfigManager     { get; }
    PopupManager       PopupManager      { get; }
}
```

- `PlatformBootstrapper.Awake()` で実装クラス `AppContext` を生成し、  
  `AppContext.Current` として登録します（1 回限り）。
- 読み取り専用（setter なし）。
- テスト時は `MockAppContext` に差し替え可能。

## 5. モジュール境界と public API

各システムは **1 つの「ファサードクラス」** を公開し、内部実装を隠蔽します。

| システム | ファサード | 主要 public API |
|--------|----------|----------------|
| キャラクター | `CharacterManager` | `LoadCharacterAsync`, `UnloadCharacter`, `CurrentVrmInstance` |
| シナリオ | `DaihonScenarioManager` | `RouteAsync`, `LoadLocalScripts`, `LoadPackageScripts` |
| トリガー | `TriggerManager` | `RegisterPlugin`, `StartMonitoring`, `OnSystemEventFired` |
| パッケージ | `PackageManager` | `InstallAsync`, `UninstallAsync`, `RestorePackagesAsync` |
| テーマ | `ThemeManager` | `LoadThemeAsync`, `ApplyTheme` |
| プラグイン | `PluginLoader` | `LoadFromDirectory` |
| LLM | `LLMBridge` | `ChatAsync` |

## 6. イベント駆動とリアクティブ更新

コンポーネント間通知は **C# event（デリゲート）** を使用します。  
Unity の `UnityEvent` は Inspector 連携が必要な場合のみ使用します。

```csharp
// イベント定義例
public event Action<bool> OnCharacterPresenceChanged;

// 発火
OnCharacterPresenceChanged?.Invoke(true);

// 購読（依存元コンポーネントの Awake / Initialize で登録）
characterManager.OnCharacterPresenceChanged += OnPresenceChanged;
```

サブスクライバは **OnDestroy でかならず登録解除** します。

## 7. 非同期処理規約

- すべての非同期メソッドは `UniTask` / `UniTask<T>` を返します。
- `async void` は `Start()` でのみ許容（`UniTaskVoid` または `.Forget()` を使用）。
- キャンセルトークンは **呼び出し元から引数で渡す** 設計を基本とします。
- MonoBehaviour が破棄される際のキャンセルは `this.GetCancellationTokenOnDestroy()` で取得します。

```csharp
// ✅ 良い例
public async UniTask SpeakAsync(string text, CancellationToken ct)
{
    await UniTask.Delay(TimeSpan.FromSeconds(1), cancellationToken: ct);
}

// ❌ 悪い例（キャンセル不能）
public async UniTask SpeakAsync(string text)
{
    await UniTask.Delay(1000); // キャンセルできない
}
```
