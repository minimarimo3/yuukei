# 12 データ永続化

## 1. 概要

アプリの状態は `Application.persistentDataPath` 以下に保存します。  
ゲームの設定は JSON、認証情報は OS の認証情報ストアで管理します。

## 2. ディレクトリ構成

```
{persistentDataPath}/
├── settings.json              アプリ設定（キャラクター・パッケージ・LLM設定）
├── pending_unload.json        次回起動時にアンロードする DLL のリスト
├── Daihon/                    ユーザー台本スクリプト
│   └── *.daihon
├── Packages/                  インストール済みパッケージ
│   └── {package-id}/
│       ├── package.json
│       ├── theme/
│       ├── plugins/
│       └── *.daihon
└── Themes/                    スタンドアロンテーマ（パッケージ外）
    └── {theme-name}/
        ├── theme.json
        └── images/
```

## 3. settings.json

```json
{
  "currentCharacterId": "a1b2c3d4-...",
  "savedCharacters": [
    {
      "id":       "a1b2c3d4-...",
      "name":     "キャラクター名",
      "filePath": "C:/Users/.../MyCharacter.vrm"
    }
  ],
  "activePackageId": "com.example.my-package",
  "installedPackages": [
    {
      "id":          "com.example.my-package",
      "name":        "マイパッケージ",
      "version":     "1.0.0",
      "installPath": "{persistentDataPath}/Packages/com.example.my-package",
      "enabled":     true
    }
  ],
  "llmSettings": {
    "provider":    "cloud",
    "endpointUrl": "https://api.openai.com/v1/chat/completions",
    "modelName":   "gpt-4o"
  }
}
```

**重要**：`llmSettings` に API キーは **含めません**。キーは `ICredentialStorage` で管理します。

## 4. ConfigManager の責務

```csharp
public class ConfigManager : MonoBehaviour
{
    public static ConfigManager Instance { get; private set; }

    public AppSettings Settings { get; private set; }

    public event Action OnSettingsChanged;

    // ── ライフサイクル ──────────────────────────────────────────────

    private void Awake()
    {
        Instance = this;
        DontDestroyOnLoad(gameObject);
        LoadSettings();
    }

    // ── 設定 I/O ───────────────────────────────────────────────────

    public void SaveSettings();   // JSON シリアライズ → settings.json
    private void LoadSettings();  // settings.json → JSON デシリアライズ

    // ── キャラクター管理 ───────────────────────────────────────────

    public void AddCharacter(CharacterData character);
    public void RemoveCharacter(string id);
    public void SetCurrentCharacter(string id);

    // ── パッケージ管理 ──────────────────────────────────────────────

    public void AddPackage(PackageInfo package);
    public void RemovePackage(string id);
    public void SetPackageEnabled(string id, bool enabled);
}
```

### 4.1 LoadSettings の詳細

```
LoadSettings()
  path = {persistentDataPath}/settings.json
  if File.Exists(path):
    text = File.ReadAllText(path)
    Settings = JsonUtility.FromJson<AppSettings>(text)
    if Settings == null:
      Settings = new AppSettings()   // デフォルト値
  else:
    Settings = new AppSettings()     // 初回起動
```

### 4.2 SaveSettings の詳細

```
SaveSettings()
  text = JsonUtility.ToJson(Settings, prettyPrint: true)
  File.WriteAllText({persistentDataPath}/settings.json, text)
  OnSettingsChanged?.Invoke()
```

## 5. 認証情報の管理

API キーなどの機密情報は `ICredentialStorage` を経由して管理します。

```
// 保存
credentialStorage.Save("llm_api_key", "sk-xxxx...")

// 読み込み
var key = credentialStorage.Load("llm_api_key")

// 削除
credentialStorage.Delete("llm_api_key")
```

**Windows**：Credential Manager（`advapi32.dll`）に `"Yuukei_{key}"` という名前で保存します。  
**その他**：メモリのみ（再起動で失われる）。

## 6. pending_unload.json

アンインストールされたプラグインの DLL パスを記録します。  
次回起動時に `PluginLoader` がこのリストを参照し、対象 DLL をロードしません。

```json
[
  "{persistentDataPath}/Packages/com.example.old-plugin/plugins/OldPlugin.dll"
]
```

`PluginLoader.LoadFromDirectory()` の冒頭でこのリストを読み込み、  
対象パスが含まれる DLL はスキップします。  
アンロード完了後（次回起動確認時）にリストからエントリを削除します。

## 7. エラーハンドリング

| 状況 | 挙動 |
|------|------|
| `settings.json` が存在しない | デフォルト設定で起動 |
| `settings.json` が壊れている（JSON パースエラー） | デフォルト設定で起動 + エラーログ |
| 書き込み権限がない | エラーログ + ユーザーに警告ポップアップ |
| VRM ファイルが移動・削除された | 起動時に警告ログ（エントリは残す） |
| パッケージディレクトリが存在しない | 警告ログ（インストール情報は残す） |
