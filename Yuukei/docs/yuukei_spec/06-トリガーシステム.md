# 06 トリガーシステム

## 1. 概要

「トリガー」はキャラクター行動の起点となるイベントです。  
`TriggerManager` が各トリガーを一元管理し、発火したイベントを `DaihonScenarioManager` へルーティングします。

## 2. アーキテクチャ

```
外部イベント（時刻・クリック・ファイルドロップ・...）
        │
        ▼
  ITriggerPlugin.OnFired（event）
        │
        ▼
  TriggerManager.OnSystemEventFired（event）
        │
        ▼
  DaihonScenarioManager.RouteAsync()
        │
        ▼
  台本シーン実行
```

## 3. TriggerManager

### 3.1 責務

- 組み込みトリガー・プラグイントリガーの登録と管理
- トリガー発火時に `OnSystemEventFired` イベントを経由して `DaihonScenarioManager` へ通知
- `Update` でポーリング型トリガーの `Tick()` を呼び出す

### 3.2 インターフェース

```csharp
public class TriggerManager : MonoBehaviour
{
    public static TriggerManager Instance { get; private set; }

    /// <summary>トリガーが発火したときに通知されるイベント。</summary>
    public event Action<TriggerPayload> OnSystemEventFired;

    /// <summary>プラグインが提供するトリガーを登録する。</summary>
    public void RegisterPlugin(ITriggerPlugin plugin);

    /// <summary>全トリガーの監視を開始する（ブートシーケンス Phase 7 で呼ぶ）。</summary>
    public void StartMonitoring();
}
```

### 3.3 TriggerPayload

```csharp
/// <summary>トリガー発火時に渡されるデータ。</summary>
public class TriggerPayload
{
    /// <summary>トリガーの識別名（台本の signal 名と一致する）。</summary>
    public string TriggerName { get; }

    /// <summary>トリガー固有のパラメータ。</summary>
    public IReadOnlyDictionary<string, object> Parameters { get; }
}
```

## 4. ITriggerPlugin コントラクト

プラグインが独自トリガーを実装するためのインターフェースです。  
詳細は [07-プラグインシステム](07-プラグインシステム.md) を参照してください。

```csharp
public interface ITriggerPlugin : IDisposable
{
    string TriggerName { get; }
    void Initialize(ITriggerContext context);
    event Action<TriggerPayload> OnFired;
}

public interface ITriggerContext
{
    // プラグインが必要とする最低限の情報のみ公開
    string PersistentDataPath { get; }
}
```

## 5. 組み込みトリガー

### 5.1 TimeTrigger（時刻トリガー）

指定した時刻・曜日に発火します。1 日に最大 1 回の発火制限があります。

**パラメータ（台本 signal の引数）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `hour` | Number | `9` |
| `minute` | Number | `0` |
| `days` | String[] | `["Mon", "Tue"]`（省略で毎日） |

**ペイロード**

| キー | 値 |
|------|----|
| `hour` | 発火した時（Number） |
| `minute` | 発火した分（Number） |

**実装詳細**

- ポーリング方式：`Update()` で現在時刻を確認
- 発火条件：時・分が一致 かつ 当日未発火（`_firedToday` フラグ）
- 日をまたいだ際（0 時）に `_firedToday` をリセット

---

### 5.2 ClickTrigger（クリックトリガー）

キャラクター上でマウス左ボタンをクリックしたときに発火します。

**ペイロード**：なし（空のパラメータ辞書）

**実装詳細**

- `CharacterDragController` がドラッグ判定を行い、クリックと判定した場合に  
  `TriggerManager` へ通知します（Observer パターン）。
- ドラッグが開始された場合は発火しません。

---

### 5.3 FileDropTrigger（ファイルドロップトリガー）

指定拡張子のファイルがウィンドウにドロップされたときに発火します。

**パラメータ（台本 signal の引数）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `extensions` | String[] | `[".txt", ".log"]` |

**ペイロード**

| キー | 値 |
|------|----|
| `filePath` | ドロップされたファイルのフルパス |
| `extension` | ファイルの拡張子（`.txt` など） |

**実装詳細**

- `FileDropRouter` がドロップイベントを受け取り、処理後に `TriggerManager` へ通知します。
- `.vrm`・`.zip` など、アプリが特別扱いする拡張子は `FileDropRouter` で先にハンドリングし、  
  `FileDropTrigger` には渡しません。

---

### 5.4 SystemEventTrigger（システムイベントトリガー）

CPU 使用率やバッテリー残量などのシステム状態変化で発火します。

**パラメータ（台本 signal の引数）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `cpuThreshold` | Number | `80`（%） |
| `batteryThreshold` | Number | `20`（%） |

**ペイロード**

| キー | 値 |
|------|----|
| `type` | `"battery"` / `"cpu"` |
| `batteryLevel` | バッテリー残量（%、Number） |

**実装詳細**

- 30 秒ごとにシステム情報をポーリング（`Tick(deltaTime)` 蓄算方式）。

---

### 5.5 ExplorerItemTrigger（エクスプローラーアイテムトリガー）（Windows のみ）

Windows エクスプローラーで指定ファイルが表示されているとき、  
キャラクターがそのアイテムの上に重なったときに発火します。

**パラメータ（台本 signal の引数）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `targetName` | String | `"important.txt"` |

**ペイロード**

| キー | 値 |
|------|----|
| `targetName` | 検出されたファイル名 |

**実装詳細**

- `ExplorerItemScanner.exe`（外部プロセス）を起動し、stdout を非同期で監視。
- 出力フォーマット：`SUCCESS:{left},{top},{width},{height}`
- エクスプローラーウィンドウが存在しない場合は無音で待機。

## 6. 発火制御と重複排除

| 課題 | 方針 |
|------|------|
| 同一シーンの二重実行 | `DaihonScenarioManager` でフラグ管理（実行中は次の発火を無視） |
| 時刻トリガーの 1 日 1 回制限 | `TimeTrigger` 内の `_firedToday` フラグ |
| 高頻度イベント（CPU, Explorer）| ポーリング間隔によるデバウンス |
