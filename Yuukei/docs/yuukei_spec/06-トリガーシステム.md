# 06 トリガーシステム

## 1. 概要

「トリガー」はキャラクター行動の起点となるイベントです。  
`TriggerManager` が各トリガーを一元管理し、発火したイベントを `DaihonScenarioManager` へルーティングします。

## 2. アーキテクチャ

```
外部イベント（時刻・クリック・ファイルドロップ・...）
        │
        ├─ ポーリング型 Trigger（Time / Tick / System / Explorer）
        ├─ リレー元コンポーネント（CharacterDragController / FileDropRouter）
        └─ ITriggerPlugin.OnFired（event）
                    │
                    ▼
           TriggerManager.Raise(payload)
                    │
                    ▼
         TriggerManager.OnSystemEventFired（event）
                    │
                    ▼
           DaihonScenarioManager.RouteAsync()
                    │
                    ▼
                台本シーン実行
```

## 3. TriggerManager

### 3.1 責務

- 組み込みトリガー・プラグイントリガーの登録と管理
- トリガー発火時に `OnSystemEventFired` イベントを経由して `DaihonScenarioManager` へ通知
- `Update` でポーリング型トリガーの `Tick(deltaTime)` を呼び出す
- リレー型イベント（クリック・ファイルドロップ）を `Raise()` で受け取る

### 3.2 インターフェース

```csharp
public class TriggerManager : MonoBehaviour
{
    /// <summary>トリガーが発火したときに通知されるイベント。</summary>
    public event Action<TriggerPayload> OnSystemEventFired;

    /// <summary>シナリオ管理へ通知するための結線を行う。</summary>
    public void Initialize(DaihonScenarioManager scenarioManager);

    /// <summary>プラグインが提供するトリガーを登録する。</summary>
    public void RegisterPlugin(ITriggerPlugin plugin);

    /// <summary>リレー元コンポーネントからトリガーを発火させる。</summary>
    public void Raise(TriggerPayload payload);

    /// <summary>全トリガーの監視を開始する（ブートシーケンス Phase 7 で呼ぶ）。</summary>
    public void StartMonitoring();
}
```

### 3.3 TriggerPayload

```csharp
/// <summary>トリガー発火時に渡されるデータ。</summary>
public class TriggerPayload
{
    /// <summary>トリガーの識別名（台本の signal 名と一致する）。</summary>
    public string TriggerName { get; }  // 例: "起動時", "クリック", "定期確認"

    /// <summary>トリガー固有のパラメータ。</summary>
    public IReadOnlyDictionary<string, object> Parameters { get; }
}
```

`DaihonScenarioManager` は `TriggerPayload` を受け取ると、`_trigger_name` と `_trigger_<key>` の一時変数へ変換して条件式評価に渡します。
変換規則は [05-スクリプトシステム](05-スクリプトシステム.md) の `SimpleVariableStore` 仕様に従います。

## 4. Trigger コントラクト

組み込みトリガーとプラグイントリガーで共有する契約です。  
詳細は [07-プラグインシステム](07-プラグインシステム.md) を参照してください。

```csharp
public interface ITriggerSource
{
    string TriggerName { get; }
    event Action<TriggerPayload> OnFired;
}

public interface IPollingTrigger : ITriggerSource
{
    void Tick(float deltaTime);
}

public interface ITriggerPlugin : ITriggerSource, IDisposable
{
    void Initialize(ITriggerContext context);
}

public interface ITriggerContext
{
    // プラグインが必要とする最低限の情報のみ公開
    string PersistentDataPath { get; }
}
```

## 5. 組み込みトリガー

`daihon_spec` の `合図:` はイベント名のみを記述します（引数は取りません）。
トリガー固有のしきい値・対象一覧はアプリ設定（UI）で管理し、台本条件からは `_trigger_<key>` で参照します。

| 用途 | `TriggerName` |
|------|---------------|
| 起動完了時のワンショット | `起動時` |
| クリック | `クリック` |
| ファイルドロップ | `ファイルドロップ` |
| 時刻一致 | `時刻` |
| システム状態変化 | `システムイベント` |
| monitorScene の定期評価 | `定期確認` |
| Explorer 重なり判定 | `エクスプローラー項目` |

### 5.1 起動時イベント

`TriggerManager.StartMonitoring()` の直後に 1 回だけ `TriggerName = "起動時"` を発火します。  
専用コンポーネントは持たず、起動シーケンスの最後で `TriggerManager` が直接発火します。

---

### 5.2 TimeTrigger（時刻トリガー）

指定した時刻・曜日に発火します。1 日に最大 1 回の発火制限があります。

**発火条件（アプリ設定）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `hour` | Number | `9` |
| `minute` | Number | `0` |
| `days` | String[] | `["Mon", "Tue"]`（省略で毎日） |

**ペイロード**

| キー | 値 |
|------|----|
| `hour` | 発火した時（Number） |
| `minute` | 発火した分（Number） |

**実装詳細**

- `TriggerName` は常に `"時刻"`。
- ポーリング方式：`Update()` で現在時刻を確認
- 発火条件：時・分が一致 かつ 当日未発火（`_firedToday` フラグ）
- 日をまたいだ際（0 時）に `_firedToday` をリセット

---

### 5.3 クリック通知（CharacterDragController リレー）

キャラクター上でマウス左ボタンをクリックしたときに発火します。

**ペイロード**：なし（空のパラメータ辞書）

**実装詳細**

- `CharacterDragController` がドラッグ判定を行い、クリックと判定した場合に  
  `TriggerManager.Raise(new TriggerPayload("クリック"))` を呼びます。
- ドラッグが開始された場合は発火しません。

---

### 5.4 ファイルドロップ通知（FileDropRouter リレー）

指定拡張子のファイルがウィンドウにドロップされたときに発火します。

**発火条件（アプリ設定）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `extensions` | String[] | `[".txt", ".log"]` |

**ペイロード**

| キー | 値 |
|------|----|
| `filePath` | ドロップされたファイルのフルパス |
| `extension` | ファイルの拡張子（`.txt` など） |

**実装詳細**

- `FileDropRouter` がドロップイベントを受け取り、処理後に `TriggerManager.Raise(...)` を呼びます。
- `TriggerName` は常に `"ファイルドロップ"`。
- `.vrm`・`.zip` など、アプリが特別扱いする拡張子は `FileDropRouter` で先にハンドリングし、  
  台本トリガーには渡しません。

---

### 5.5 SystemEventTrigger（システムイベントトリガー）

CPU 使用率やバッテリー残量などのシステム状態変化で発火します。

**発火条件（アプリ設定）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `cpuThreshold` | Number | `80`（%） |
| `batteryThreshold` | Number | `20`（%） |

**ペイロード**

| キー | 値 |
|------|----|
| `type` | `"battery"` / `"cpu"` |
| `batteryLevel` | バッテリー残量（%、Number） |

**実装詳細**

- `TriggerName` は常に `"システムイベント"`。
- 30 秒ごとにシステム情報をポーリング（`Tick(deltaTime)` 蓄算方式）。
- Windows環境では `SystemInfo.batteryLevel` が機能しないため、P/Invoke で `GetSystemPowerStatus` を呼び出し正確なバッテリー残量を取得する。

---

### 5.6 TickTrigger（定期発火トリガー）

`monitorScene` のような「一定の合図なしに状態条件だけで評価されるシーン」を定期的にチェックするためのトリガーです。

**発火条件（アプリ設定）**：なし

**ペイロード**：なし（空のパラメータ辞書）

**実装詳細**

- `TriggerName` は常に `"定期確認"`。
- `Update()` 内で一定間隔（例: 1 秒ごと）に発火します。
- これにより、他のトリガーが発火していない間も `DaihonScenarioManager.RouteAsync` が定期的に呼び出され、「時間が18時になった」などの純粋な条件の `monitorScene` が適切に評価・実行されます。

---

### 5.7 ExplorerItemTrigger（エクスプローラーアイテムトリガー）（Windows のみ）

Windows エクスプローラーで指定ファイルが表示されているとき、  
キャラクターがそのアイテムの上に重なったときに発火します。

**発火条件（アプリ設定）**

| パラメータ | 型 | 例 |
|-----------|-----|-----|
| `targetName` | String | `"important.txt"` |

**ペイロード**

| キー | 値 |
|------|----|
| `targetName` | 検出されたファイル名 |

**実装詳細**

- `TriggerName` は常に `"エクスプローラー項目"`。
- `ExplorerItemScanner.exe`（外部プロセス）を起動し、stdout を非同期で監視。
- 出力フォーマット：`SUCCESS:{left},{top},{width},{height}`
- エクスプローラーウィンドウが存在しない場合は無音で待機。

## 6. 発火制御と重複排除

| 課題 | 方針 |
|------|------|
| 同一シーンの二重実行 | `DaihonScenarioManager` でフラグ管理（実行中は次の発火を無視） |
| 時刻トリガーの 1 日 1 回制限 | `TimeTrigger` 内の `_firedToday` フラグ |
| 高頻度イベント（CPU, Explorer）| ポーリング間隔によるデバウンス |
