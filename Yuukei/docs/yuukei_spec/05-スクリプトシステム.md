# 05 スクリプトシステム（台本 DSL）

## 1. 概要

「台本（Daihon）」は、キャラクターの振る舞いを記述するためのドメイン固有言語（DSL）です。  
ANTLR4 で生成されたパーサー（`Daihon.dll`）を用いて解析・実行します。

台本 DSL の文法詳細は [`docs/daihon_spec/`](../daihon_spec/index.md) を参照してください。  
本ドキュメントでは、Unity 側の実行基盤を定義します。

## 2. コンポーネント構成

```
DaihonScenarioManager       シナリオ管理・トリガーとのルーティング
  └─ DaihonRunner            スクリプト文字列のパース・実行
       ├─ YuukeiActionHandler IUniTaskActionHandler 実装（関数の橋渡し）
       │    └─ DaihonFunctionRegistry  プラグイン関数の検索
       └─ SimpleVariableStore  変数ストア
```

## 3. DaihonRunner（実行エンジン）

### 3.1 責務

ANTLR4 パーサーを呼び出してスクリプトを解析・実行する静的ファサードです。  
UniTask と Task の変換（`UniTaskActionAdapter`）を内部で行います。

### 3.2 API

```csharp
public static class DaihonRunner
{
    /// <summary>スクリプト全体を実行する。</summary>
    public static async UniTask RunAsync(
        string scriptText,
        IUniTaskActionHandler actionHandler,
        IVariableStore variableStore);

    /// <summary>指定シーンを実行する。</summary>
    public static async UniTask RunSceneAsync(
        string scriptText,
        string startSceneName,
        IUniTaskActionHandler actionHandler,
        IVariableStore variableStore);

    /// <summary>スクリプトのメタデータ（シーン・シグナル・条件）を抽出する。</summary>
    public static DaihonFileMetadata ExtractMetadata(string scriptText);
}
```

### 3.3 メタデータ型定義

```csharp
/// <summary>台本ファイルのメタデータ。ルーティングに使用。</summary>
public class DaihonFileMetadata
{
    /// <summary>ファイルパス。</summary>
    public string FilePath { get; set; }

    /// <summary>イベント名（## の後のテキスト）。</summary>
    public string EventName { get; set; }

    /// <summary>前提条件の式（ある場合）。</summary>
    public string PreconditionExpression { get; set; }

    /// <summary>シーンごとのメタデータ。</summary>
    public List<SceneMetadata> Scenes { get; set; }
}

public class SceneMetadata
{
    public string SceneName { get; set; }

    /// <summary>合図のシグナル名リスト（null = シグナルなし）。</summary>
    public List<string> SignalNames { get; set; }

    /// <summary>条件式の文字列表現（null = 条件なし）。</summary>
    public string ConditionExpression { get; set; }

    /// <summary>シーンの分類（ルーティング時に使用）。</summary>
    public SceneCategory Category { get; set; }
}

public enum SceneCategory
{
    Target,   // シグナル指定あり
    Monitor,  // シグナルなし・条件あり
    Default   // シグナルなし・条件なし
}
```

### 3.4 Daihon.dll 共有型定義

`Daihon.dll`（ANTLR パーサー）で定義され、Unity 側で使用される型です。

```csharp
/// <summary>台本 DSL の値を表す型。Number / String / Boolean のいずれか。</summary>
public class DaihonValue
{
    public DaihonValueType Type { get; }
    public double AsNumber();
    public string AsString();
    public bool AsBoolean();

    public static DaihonValue FromNumber(double value);
    public static DaihonValue FromString(string value);
    public static DaihonValue FromBoolean(bool value);
    public static DaihonValue None { get; }  // 戻り値なしを示す
}

public enum DaihonValueType { Number, String, Boolean, None }

/// <summary>台本の関数呼び出し・セリフ表示を処理するハンドラ。</summary>
public interface IUniTaskActionHandler
{
    UniTask ShowDialogueAsync(string text);
    UniTask<DaihonValue> CallFunctionAsync(
        string functionName,
        IReadOnlyList<DaihonValue> positionalArgs,
        IReadOnlyDictionary<string, DaihonValue> namedArgs);
}

/// <summary>台本実行中の変数を管理するストア。</summary>
public interface IVariableStore
{
    DaihonValue GetVariable(string name);
    void SetVariable(string name, DaihonValue value);
    bool HasVariable(string name);
    void RegisterDynamicGetter(string name, Func<DaihonValue> getter);
    void ClearTemporaryVariables();
}
```

### 3.5 実行フロー

```
RunSceneAsync(scriptText, sceneName, handler, store)
  1. DaihonLexer / DaihonParser でパース
  2. UniTaskActionAdapter でハンドラを Task に変換
  3. 指定シーンを検索して実行開始
  4. EventEndException / SceneEndException はキャッチしてフロー制御に使用
  5. パースエラーは Debug.LogError で記録
```

## 4. DaihonScenarioManager（シナリオ管理）

### 4.1 責務

- 台本ファイルのスキャン・メタデータのキャッシュ
- トリガーペイロードを受け取り、一致するシーンを選択・実行
- パッケージごとの変数ストアの管理

### 4.2 シナリオ分類

台本ファイルのシーンは 3 種類に分類されます。

| 分類 | 条件 | 評価タイミング |
|------|------|----------------|
| `targetScene` | シグナル指定あり | 対応トリガー発火時 |
| `monitorScene` | シグナルなし・条件あり | 全トリガー発火後に常時評価 |
| `defaultScene` | シグナルなし・条件なし | targetScene が 1 つも一致しなかった場合にランダム実行 |

### 4.3 ルーティングアルゴリズム

RouteAsync はトリガーペイロードに一致するシーンを**ファイル横断で**検索します。
一致するシーンが見つかった場合、そのシーンを含むイベントファイル全体を `DaihonRunner.RunSceneAsync()` で実行します。
1 ファイル内の複数シーン順次実行は [daihon_spec §2.6.5](../daihon_spec/02-台本の構造.md#265-デフォルトシーンと複数シーンの同時成立) のルールに従います。

```
RouteAsync(TriggerPayload payload)
  1. payload.TriggerName に一致する targetScene を列挙
  2. 各 targetScene の条件を DaihonRunner で評価
  3. 条件が真の targetScene があれば → そのシーンを実行して終了
  4. targetScene が存在するが全て条件が偽 → defaultScene をランダム選択して実行
  5. targetScene が 0 件 → monitorScene を条件評価して実行
```

### 4.4 インターフェース

```csharp
public class DaihonScenarioManager : MonoBehaviour
{
    // 依存（Inspector から注入）
    [SerializeField] private YuukeiActionHandler actionHandler;

    /// <summary>persistentDataPath/Daihon/ のスクリプトをロードする。</summary>
    public void LoadLocalScripts();

    /// <summary>パッケージのスクリプトをロードする。</summary>
    public void LoadPackageScripts(string packagePath);

    /// <summary>トリガーペイロードを受け取り、適切なシーンを実行する。</summary>
    public async UniTask RouteAsync(TriggerPayload payload, CancellationToken ct = default);
}
```

### 4.5 組み込み動的変数

シナリオ実行中に参照できる読み取り専用の動的変数です。

| 変数名 | 型 | 内容 |
|--------|-----|------|
| `年` | Number | 現在の年（西暦） |
| `月` | Number | 現在の月（1〜12） |
| `日` | Number | 現在の日（1〜31） |
| `時` | Number | 現在の時（0〜23） |
| `分` | Number | 現在の分（0〜59） |
| `秒` | Number | 現在の秒（0〜59） |
| `ミリ秒` | Number | 現在のミリ秒（0〜999） |
| `曜日` | String | 曜日（"月"〜"日"） |
| `週` | Number | その月の第何週目か（1〜5） |

## 5. YuukeiActionHandler（組み込み関数）

### 5.1 責務

`IUniTaskActionHandler` を実装し、台本 DSL の関数呼び出しを Unity コンポーネントに委譲します。  
未知の関数名は `DaihonFunctionRegistry` に委譲します。

### 5.2 組み込み関数一覧

| 関数名 | 引数 | 戻り値 | 処理内容 |
|--------|------|--------|----------|
| `表情` | 表情名: String | None | VRM 表情ウェイトを 1.0 に設定 |
| `動作` | モーション名: String | None | VRMA 再生 / "停止" で停止 |
| `待つ` / `w` | 秒数: Number | None | 指定秒数だけ待機 |
| `ランダム` | 最小値, 最大値: Number | Number | 範囲内のランダム浮動小数点数を返す |
| `現在の天気` | なし | String | 現在の天気を返す（TODO: 実 API 連携） |
| `LLM` | プロンプト: String | String | LLMBridge.ChatAsync() を呼び出す |
| `移動` | x, y: Number（0〜1000） | None | ビューポート座標へ移動 |
| `選択肢表示` | 選択肢: String... | None | 選択肢 UI を表示し、ユーザーの選択を待機する。選択結果は変数 `選択結果` に文字列として代入される |

### 5.3 インターフェース

```csharp
public class YuukeiActionHandler : MonoBehaviour, IUniTaskActionHandler
{
    // 依存（Inspector から注入）
    [SerializeField] private CharacterManager         characterManager;
    [SerializeField] private VRMLipSync               lipSync;
    [SerializeField] private VrmaPlayer               vrmaPlayer;
    [SerializeField] private FloatingMotionController floatingMotion;
    [SerializeField] private SpeechBubbleController   speechBubble;
    [SerializeField] private LLMBridge                llmBridge;

    /// <summary>セリフを表示し口パクを再生する。完了まで await する。</summary>
    public async UniTask ShowDialogueAsync(string text);

    /// <summary>関数名に応じてディスパッチする。未知の関数は DaihonFunctionRegistry に委譲。</summary>
    public async UniTask<DaihonValue> CallFunctionAsync(
        string functionName,
        IReadOnlyList<DaihonValue> positionalArgs,
        IReadOnlyDictionary<string, DaihonValue> namedArgs);
}
```

## 6. DaihonFunctionRegistry（プラグイン関数レジストリ）

### 6.1 責務

プラグインが提供するカスタム台本関数を登録・検索するための静的レジストリです。

### 6.2 インターフェース

```csharp
public static class DaihonFunctionRegistry
{
    /// <summary>カスタム関数を登録する。同名関数は上書き。</summary>
    public static void Register(IDaihonFunction function);

    /// <summary>関数名で検索する。</summary>
    public static bool TryGet(string name, out IDaihonFunction function);

    /// <summary>登録済み関数の一覧を返す。</summary>
    public static IReadOnlyCollection<string> GetRegisteredNames();
}
```

## 7. SimpleVariableStore（変数ストア）

### 7.1 責務

台本実行中の変数を管理します。スクリプトファイルごとに 1 つのインスタンスが作成されます。

### 7.2 設計規則

- 変数の型（Number / String / Boolean）は **最初の代入時に確定** し、後から変更できません。
- `_` プレフィックスの変数はイベント終了時に自動的にクリアされます（一時変数）。
- 動的ゲッター（読み取り専用の計算値）を登録できます。

### 7.3 インターフェース

```csharp
public class SimpleVariableStore : IVariableStore
{
    /// <summary>動的ゲッターを登録する（年・月・日などの組み込み変数）。</summary>
    public void RegisterDynamicGetter(string name, Func<DaihonValue> getter);

    /// <summary>イベント終了時に一時変数（_ プレフィックス）をクリアする。</summary>
    public void ClearTemporaryVariables();
}
```

## 8. スクリプトファイルの配置

| 場所 | パス | 管理者 |
|------|------|--------|
| ユーザースクリプト | `{persistentDataPath}/Daihon/*.daihon` | DaihonScenarioManager |
| パッケージスクリプト | `{packageInstallPath}/*.daihon` | PackageManager 経由 |

対応拡張子: `.daihon`、`.txt`（どちらも同じフォーマット）
