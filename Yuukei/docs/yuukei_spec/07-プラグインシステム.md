# 07 プラグインシステム

## 1. 概要

C# DLL プラグインによる機能拡張を提供します。  
プラグインは `ITriggerPlugin` または `IDaihonFunction` を実装することで、  
新しいトリガーや台本関数を追加できます。

## 2. プラグインコントラクト

プラグイン作成者が参照するインターフェース群です（`Yuukei.ModAPI.dll` として配布）。

```csharp
namespace Yuukei.ModAPI
{
    // ─── トリガープラグイン ───────────────────────────────────────────

    /// <summary>カスタムトリガーを実装するためのインターフェース。</summary>
    public interface ITriggerPlugin : IDisposable
    {
        /// <summary>台本の signal 名と一致するトリガー識別名。</summary>
        string TriggerName { get; }

        /// <summary>プラグインの初期化。アプリ起動時に 1 回呼ばれる。</summary>
        void Initialize(ITriggerContext context);

        /// <summary>トリガーが発火したときに通知するイベント。</summary>
        event Action<TriggerPayload> OnFired;
    }

    /// <summary>プラグインに渡されるコンテキスト。必要最低限のアクセスのみ公開する。</summary>
    public interface ITriggerContext
    {
        string PersistentDataPath { get; }
    }

    // ─── 台本関数プラグイン ──────────────────────────────────────────

    /// <summary>台本 DSL にカスタム関数を追加するためのインターフェース。</summary>
    public interface IDaihonFunction
    {
        /// <summary>台本内での関数呼び出し名（例: "天気取得"）。</summary>
        string FunctionName { get; }

        /// <summary>台本から呼び出されたときに実行される処理。</summary>
        UniTask<DaihonValue> CallAsync(
            IReadOnlyList<DaihonValue> positionalArgs,
            IReadOnlyDictionary<string, DaihonValue> namedArgs);
    }

    // ─── 共通型 ─────────────────────────────────────────────────────

    public class TriggerPayload
    {
        public string TriggerName { get; }
        public IReadOnlyDictionary<string, object> Parameters { get; }

        public TriggerPayload(string triggerName, Dictionary<string, object> parameters = null)
        {
            TriggerName = triggerName;
            Parameters  = parameters ?? new Dictionary<string, object>();
        }
    }
}
```

## 3. PluginLoader

### 3.1 責務

指定ディレクトリの DLL をスキャンし、`ITriggerPlugin` / `IDaihonFunction` の  
実装クラスを発見してインスタンス化・登録します。

### 3.2 インターフェース

```csharp
public class PluginLoader : MonoBehaviour
{
    public static PluginLoader Instance { get; private set; }

    // 依存（Inspector から注入）
    [SerializeField] private TriggerManager triggerManager;

    /// <summary>指定ディレクトリ以下の DLL をロードし、プラグインを登録する。</summary>
    public void LoadFromDirectory(string pluginsDirectory);
}
```

### 3.3 ロード処理の詳細

```
LoadFromDirectory(dir)
  foreach dll in Directory.GetFiles(dir, "*.dll"):
    try:
      assembly = Assembly.LoadFrom(dll)
      foreach type in assembly.GetTypes():
        if type implements ITriggerPlugin:
          plugin = Activator.CreateInstance(type)
          plugin.Initialize(triggerContext)
          triggerManager.RegisterPlugin(plugin)
        if type implements IDaihonFunction:
          func = Activator.CreateInstance(type)
          DaihonFunctionRegistry.Register(func)
    catch ReflectionTypeLoadException:
      Debug.LogError("[PluginLoader] ..." + dll)
      continue（他の DLL のロードを継続）
```

### 3.4 アンロード

CLR の制限により、ロード済み DLL を実行中にアンロードすることはできません。

- アンインストール時には `pending_unload.json` に DLL パスを記録します。
- 次回起動時に `PluginLoader` が `pending_unload.json` を確認し、対象 DLL をロードしません。

## 4. セキュリティ

### 4.1 プラグインインストール時の警告

パッケージに `plugins/` ディレクトリが含まれる場合、`FileDropRouter` が  
インストール前にユーザーへセキュリティ警告ダイアログを表示します。

```
⚠ 警告
このパッケージには実行可能なプラグイン（DLL）が含まれています。
悪意のあるプラグインはシステムに損害を与える可能性があります。
信頼できる提供元のパッケージのみをインストールしてください。

[ インストール ] [ キャンセル ]
```

ユーザーが確認した場合のみインストールを実行します。

### 4.2 サンドボックス

プラグインは通常の .NET アセンブリとして動作するため、  
フルアクセスが可能です（Minecraft モデルの「ユーザー自己責任」方針）。  
将来的には `AssemblyLoadContext` を利用したアンロード対応を検討します。

## 5. プラグイン開発ガイド

### 5.1 カスタムトリガーの実装例

```csharp
using Yuukei.ModAPI;

public class WebhookTrigger : ITriggerPlugin
{
    public string TriggerName => "webhook";

    public event Action<TriggerPayload> OnFired;

    private HttpListener _listener;

    public void Initialize(ITriggerContext context)
    {
        _listener = new HttpListener();
        _listener.Prefixes.Add("http://localhost:8765/");
        _listener.Start();
        _ = ListenAsync();
    }

    private async Task ListenAsync()
    {
        while (_listener.IsListening)
        {
            var ctx = await _listener.GetContextAsync();
            OnFired?.Invoke(new TriggerPayload("webhook",
                new Dictionary<string, object> { ["url"] = ctx.Request.Url.ToString() }));
            ctx.Response.Close();
        }
    }

    public void Dispose() => _listener?.Stop();
}
```

### 5.2 カスタム台本関数の実装例

```csharp
using Yuukei.ModAPI;

public class GetWeatherFunction : IDaihonFunction
{
    public string FunctionName => "天気取得";

    public async UniTask<DaihonValue> CallAsync(
        IReadOnlyList<DaihonValue> positionalArgs,
        IReadOnlyDictionary<string, DaihonValue> namedArgs)
    {
        var city = positionalArgs.Count > 0 ? positionalArgs[0].AsString() : "Tokyo";
        var weather = await WeatherApi.FetchAsync(city);
        return DaihonValue.FromString(weather);
    }
}
```

### 5.3 DLL の配置

プラグイン DLL はパッケージ内の `plugins/` フォルダに配置します。

```
my-package.zip
├── package.json
└── plugins/
    └── MyPlugin.dll
```
