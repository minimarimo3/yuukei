# Yuukei 仕様書

本ドキュメントは、Unityデスクトップマスコット「Yuukei」をゼロから再実装するための設計書です。

## 目的

- AIコーディングアシスタントが迷いなく実装できる明確な仕様を提供する
- バグが混入しにくいクリーンなアーキテクチャを定義する
- 拡張性を保ちながら保守しやすい設計を示す

## アーキテクチャ方針

**Pure DI 主軸のハイブリッド構成** を採用します。

| 層 | 手法 | 適用範囲 |
|----|------|----------|
| 基本 | Pure DI | 全コンポーネントの依存注入 |
| 結合点 | `PlatformBootstrapper` | 起動時のインスタンス組み立て |
| 共通基盤 | 軽量 Service Locator | アプリ全体で共有される少数のサービス |

- **コンストラクタ・SerializeField でシグネチャから依存が読み取れる**設計を優先します。
- **複雑な DI コンテナは導入しません**。
- Service Locator は `IAppContext`（後述）のみに限定し、乱用しません。

## 実装に関する共通規約

- **非同期処理** はすべて `UniTask` を使用します（`async void` 禁止）。
- **MonoBehaviour の依存** は `[SerializeField]` またはコンストラクタで明示します。
- **例外** は境界（Platform 層・外部 I/O）でのみキャッチし、内部ロジックでは伝播させます。
- **ログ** は `[ClassName]` プレフィックスを付けて `Debug.Log/Warning/Error` で出力します。
- **null 安全** のため、参照が設定されていない場合は早期リターンし警告を出します。
