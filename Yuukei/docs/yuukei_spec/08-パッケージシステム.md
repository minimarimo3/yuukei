# 08 パッケージシステム

## 1. 概要

「パッケージ」はテーマ・プラグイン・台本スクリプトをまとめた ZIP アーカイブです。  
ドロップするだけでインストールでき、ユーザーがエディタなしで機能を拡張できます。

## 2. パッケージフォーマット

```
my-package.zip
├── package.json        （必須）メタデータ
├── theme/              （任意）テーマデータ
│   ├── theme.json
│   └── images/
├── plugins/            （任意）C# プラグイン DLL
│   └── *.dll
└── *.daihon            （任意）台本スクリプト
```

### 2.1 package.json

```json
{
  "id":            "com.example.my-package",
  "name":          "マイパッケージ",
  "version":       "1.0.0",
  "author":        "作者名",
  "description":   "パッケージの説明",
  "minAppVersion": "0.1.0"
}
```

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `id` | ✅ | 一意識別子（逆ドメイン推奨） |
| `name` | ✅ | 表示名 |
| `version` | ✅ | セマンティックバージョニング |
| `author` | ❌ | 作者名 |
| `description` | ❌ | 説明文 |
| `minAppVersion` | ❌ | 必要な最小アプリバージョン |

## 3. PackageManager

### 3.1 責務

- パッケージの ZIP インストール・アンインストール
- インストール済みパッケージの起動時復元
- `PluginLoader`・`ThemeManager`・`DaihonScenarioManager` への委譲

### 3.2 インターフェース

```csharp
public class PackageManager : MonoBehaviour
{
    // 依存（Inspector から注入）
    [SerializeField] private PluginLoader          pluginLoader;
    [SerializeField] private ThemeManager          themeManager;
    [SerializeField] private DaihonScenarioManager scenarioManager;
    [SerializeField] private ConfigManager         configManager;

    /// <summary>ZIP ファイルからパッケージをインストールする。</summary>
    public async UniTask InstallAsync(string zipPath, CancellationToken ct = default);

    /// <summary>指定 ID のパッケージをアンインストールする。</summary>
    public async UniTask UninstallAsync(string packageId, CancellationToken ct = default);

    /// <summary>起動時にインストール済みパッケージを復元する。</summary>
    public async UniTask RestorePackagesAsync(CancellationToken ct = default);
}
```

### 3.3 インストール処理

```
InstallAsync(zipPath)
  1. ZIP に package.json が含まれているか検証
  2. package.json をパースして PackageInfo を生成
  3. plugins/ ディレクトリが含まれる場合 → セキュリティ警告を表示
     └─ ユーザーがキャンセルしたら中断
  4. {persistentDataPath}/Packages/{id}_{version}/ に ZIP を展開（同一 ID の DLL ロック問題回避のためバージョン付きディレクトリにする）
  5. theme/ が含まれる場合 → ThemeManager.LoadThemeAsync()
  6. plugins/ が含まれる場合 → PluginLoader.LoadFromDirectory()
  7. *.daihon が含まれる場合 → DaihonScenarioManager.LoadPackageScripts()
  8. ConfigManager.Settings.installedPackages にエントリを追加して保存
```

### 3.4 アンインストール処理

```
UninstallAsync(packageId)
  1. PackageInfo を ConfigManager から取得
  2. plugins/ がある場合 → pending_unload.json に DLL パスを記録（次回起動時にロード除外）
  3. ConfigManager.Settings.installedPackages からエントリを削除して保存（この時点で復元対象外）
  4. installPath のディレクトリ削除を試行
     - 成功: 完了
     - 失敗（IOException / UnauthorizedAccessException）:
       警告ログを出して残骸を許容し、次回起動時 `CleanupPendingUninstallAsync()` で再試行
```

### 3.5 復元処理

```
RestorePackagesAsync()
  0. CleanupPendingUninstallAsync()  // pending_unload.json を参照して残骸ディレクトリ削除を再試行
  foreach PackageInfo in configManager.Settings.installedPackages where enabled:
    installPath = {persistentDataPath}/Packages/{id}_{version}/
    if Directory.Exists(installPath):
      await InstallContentAsync(installPath)  // ZIP 展開済みのため展開不要
    else:
      Debug.LogWarning("パッケージが見つかりません: " + id)
      // インストール情報は残す（ユーザーが手動で再インストール可能）
```

## 4. ZIP の判定ロジック

`FileDropRouter` が `.zip` ファイルをドロップされたとき、以下の順で判定します。

```
1. ZIP 内に package.json があるか?
   → Yes: PackageManager.InstallAsync()
   → No:
     2. ZIP 内に theme.json があるか?
        → Yes: ThemeManager.LoadThemeFromZipAsync()
        → No: 未対応 ZIP として警告
```

## 5. パッケージの保存先

| データ | パス |
|--------|------|
| インストール先 | `{persistentDataPath}/Packages/{package-id}_{version}/` |
| インストール情報 | `{persistentDataPath}/settings.json` の `installedPackages` |
| アンロード待ち | `{persistentDataPath}/pending_unload.json` |
