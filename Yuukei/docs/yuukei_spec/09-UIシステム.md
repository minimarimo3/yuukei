# 09 UI システム

## 1. 概要

UI は UI Toolkit（UXML/USS）をベースに構築します。  
テーマシステムにより、コードを変更せずに外観をカスタマイズできます。

## 2. 設定 UI（SettingsUIManager）

### 2.1 タブ構成

| タブ ID | 内容 |
|---------|------|
| `Tab_General` | 一般設定（起動・ウィンドウ等） |
| `Tab_Character` | キャラクター選択・管理 |
| `Tab_Script` | 台本ファイル管理 |
| `Tab_Prop` | キャラクター小物（将来拡張） |
| `Tab_Theme` | テーマ選択・管理 |
| `Tab_Mod` | プラグイン管理 |
| `Tab_Package` | パッケージ管理 |
| `Tab_Market` | マーケット（将来拡張） |
| `Tab_About` | バージョン情報 |

### 2.2 インターフェース

```csharp
public class SettingsUIManager : MonoBehaviour
{
    /// <summary>設定パネルを表示する（システムトレイの「設定」から呼ばれる）。</summary>
    public void ShowSettings();

    /// <summary>設定パネルを非表示にする。</summary>
    public void HideSettings();
}
```

### 2.3 パネルドラッグ

`PanelDragManipulator`（PointerManipulator）を使ってパネルをドラッグ移動できます。

```csharp
/// <summary>UI Toolkit パネルをドラッグ可能にするマニピュレーター。</summary>
public class PanelDragManipulator : PointerManipulator
{
    // 最初のドラッグ時に Flex レイアウトから Absolute に切り替え
    // PointerCapture でマウスが速く動いても追従
}
```

## 3. ポップアップシステム

### 3.1 PopupManager

```csharp
public class PopupManager : MonoBehaviour
{
    public enum PopupType { General, Info, Warning, Error, Glitch }

    /// <summary>メッセージポップアップを表示する。</summary>
    public void ShowPopup(PopupType type, string message);

    /// <summary>確認ダイアログを表示する（OK / キャンセル）。</summary>
    public void ShowWarning(
        string message,
        Action onConfirm,
        Action onCancel);
}
```

### 3.2 PopupController

個々のポップアップの動作を担当します。

```csharp
public class PopupController : MonoBehaviour
{
    // テーマ対応：背景・アイコン画像を動的にロード
    // 閉じるボタン：ホバー・押下状態に対応
    // Confirm / Cancel コールバック
    // Sprite の動的生成リソースを OnDestroy でクリーンアップ
}
```

## 4. テーマシステム（ThemeManager）

### 4.1 テーマフォーマット

テーマは `theme.json` と画像ファイルで構成されます。

```json
{
  "name": "テーマ名",
  "popup": {
    "General":  { "backgroundImage": "images/bg_general.png",  "backgroundDrawMode": "Sliced" },
    "Info":     { "backgroundImage": "images/bg_info.png",     "backgroundDrawMode": "Sliced" },
    "Warning":  { "backgroundImage": "images/bg_warning.png",  "backgroundDrawMode": "Sliced" },
    "Error":    { "backgroundImage": "images/bg_error.png",    "backgroundDrawMode": "Sliced" },
    "Glitch":   { "backgroundImage": "images/bg_glitch.png",   "backgroundDrawMode": "Sliced" }
  },
  "speechBubble": {
    "backgroundImage":  "images/bubble_bg.png",
    "backgroundDrawMode": "Sliced",
    "contentPadding":   { "top": 10, "right": 14, "bottom": 10, "left": 14 },
    "iconImage":        "images/icon.png",
    "fontColorHtml":    "#333333",
    "closeButton": {
      "NormalImage":  "images/close_normal.png",
      "HoverImage":   "images/close_hover.png",
      "PressedImage": "images/close_pressed.png",
      "offset":       { "x": -8, "y": 8 },
      "size":         { "x": 24, "y": 24 }
    }
  }
}
```

### 4.2 ThemeManager インターフェース

```csharp
public class ThemeManager : MonoBehaviour
{
    /// <summary>ディレクトリからテーマを読み込んで適用する。</summary>
    public async UniTask LoadThemeAsync(string themeDirPath, CancellationToken ct = default);

    /// <summary>ZIP ファイルからテーマを読み込んで適用する。</summary>
    public async UniTask LoadThemeFromZipAsync(string zipPath, CancellationToken ct = default);

    /// <summary>読み込み済みテーマを UI に適用する。</summary>
    public void ApplyTheme(ThemeSettings theme);
}
```

### 4.3 テーマの保存先

| データ | パス |
|--------|------|
| テーマディレクトリ | `{persistentDataPath}/Themes/{theme-name}/` |

## 5. FileDropRouter（ファイルルーティング）

### 5.1 責務

ウィンドウにドロップされたファイルを拡張子に応じた処理にルーティングします。

### 5.2 ルーティングテーブル

| 拡張子 | ルーティング先 | 処理 |
|--------|--------------|------|
| `.vrm` | ConfigManager → CharacterManager | キャラクター登録・切り替え |
| `.bundle` | ConfigManager → CharacterManager | AssetBundle キャラクター（将来対応） |
| `.zip`（package.json あり） | PackageManager | パッケージインストール |
| `.zip`（theme.json あり）| ThemeManager | テーマ読み込み |
| `.daihon` / `.txt` | DaihonScenarioManager | `{persistentDataPath}/Daihon/` にファイルをコピーした後、`DaihonScenarioManager.LoadLocalScripts()` を呼んでリロード |

### 5.3 インターフェース

```csharp
public class FileDropRouter : MonoBehaviour
{
    // 依存（Inspector から注入）
    [SerializeField] private CharacterManager      characterManager;
    [SerializeField] private PackageManager        packageManager;
    [SerializeField] private ThemeManager          themeManager;
    [SerializeField] private DaihonScenarioManager scenarioManager;
    [SerializeField] private ConfigManager         configManager;
    [SerializeField] private PopupManager          popupManager;
}
```

## 6. UILayoutUtility

ストレッチアンカーの `RectTransform` に対してパディングを適用するユーティリティです。

```csharp
public static class UILayoutUtility
{
    /// <summary>
    /// ストレッチアンカーの RectTransform に offsetMin / offsetMax でパディングを適用する。
    /// </summary>
    public static void ApplyPadding(RectTransform targetRect, PaddingDef padding);
}

[Serializable]
public struct PaddingDef
{
    public float top;
    public float right;
    public float bottom;
    public float left;
}
```

## 7. CharacterDependentVisibility

キャラクターが存在するかどうかに応じて GameObject の表示を切り替えます。  
「キャラクターがいない場合はドロップのヒントを表示する」などに使用します。

```csharp
public class CharacterDependentVisibility : MonoBehaviour
{
    [SerializeField] private CharacterManager characterManager;

    /// <summary>キャラクター存在時に表示する GameObject。</summary>
    [SerializeField] private GameObject[] showWhenPresent;

    /// <summary>キャラクター不在時に表示する GameObject。</summary>
    [SerializeField] private GameObject[] showWhenAbsent;
}
```
