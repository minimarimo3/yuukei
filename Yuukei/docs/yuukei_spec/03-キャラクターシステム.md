# 03 キャラクターシステム

## 1. 概要

VRM 1.0 形式のキャラクターモデルの読み込み・配置・操作を管理するサブシステムです。  
`CharacterManager` が VRM ライフサイクルの唯一のオーナーとなります。

## 2. CharacterManager

### 2.1 責務

- VRM ファイルの非同期ロード／アンロード
- 現在の `Vrm10Instance` の保持と公開
- キャラクター存在状態の変化イベント通知
- アウトライン描画レイヤーの設定

### 2.2 インターフェース

```csharp
public class CharacterManager : MonoBehaviour
{
    // 依存（Inspector から注入）
    [SerializeField] private Transform          characterRoot;
    [SerializeField] private ObjectToBottomRight positioner;
    [SerializeField] private VRMLipSync         lipSync;

    // 公開状態
    public Vrm10Instance CurrentVrmInstance { get; private set; }

    // イベント
    public event Action<bool> OnCharacterPresenceChanged;

    // --- API ---

    /// <summary>
    /// 指定パスの VRM を読み込んでキャラクターを差し替える。
    /// 既存キャラクターがいる場合は先に破棄する。
    /// </summary>
    public async UniTask LoadCharacterAsync(string vrmPath, CancellationToken ct = default);

    /// <summary>
    /// 現在のキャラクターを破棄する。
    /// </summary>
    public void UnloadCharacter();

    /// <summary>
    /// 起動時に settings.json の currentCharacterId からキャラクターを復元する。
    /// </summary>
    public async UniTask RestoreCharacterAsync(CancellationToken ct = default);
}
```

### 2.3 ロード処理の詳細

```
LoadCharacterAsync(vrmPath)
  ├─ UnloadCharacter()         既存キャラクターの破棄
  ├─ Vrm10.LoadPathAsync()     非同期 VRM 読み込み
  ├─ instance.transform を characterRoot の子にアタッチ
  ├─ アウトライン描画レイヤー設定（renderingLayerMask = 256）
  ├─ クリック判定用の BoxCollider を動的に生成してアタッチ
  ├─ lipSync.InvalidateCache() キャッシュのリセット
  ├─ UniTask.Yield(PlayerLoopTiming.PostLateUpdate) で 1 フレーム待機（Bounds 確定待ち）
  ├─ positioner.PositionAtBottomRight() 配置計算
  └─ OnCharacterPresenceChanged?.Invoke(true)
```

**定数**

| 定数 | 値 | 意味 |
|------|----|------|
| `OutlineRenderingLayerMask` | `256` | アウトライン描画用レイヤーマスク |

## 3. ObjectToBottomRight（キャラクター配置）

### 3.1 責務

キャラクターをスクリーン右下に配置します。  
キャラクターサイズが変化した場合（別の VRM を読み込んだ場合など）に再計算します。

### 3.2 アルゴリズム

```
PositionAtBottomRight()
  1. ビューポート右下 (1, 0) をワールド座標に変換
  2. 全 Renderer の Bounds を集計してキャラクターの AABB を求める
  3. AABB の右下端が目標座標になるよう transform.position を算出
  4. FloatingMotionController.SetAnchorPosition() でアンカーを更新
```

### 3.3 インターフェース

```csharp
public class ObjectToBottomRight : MonoBehaviour
{
    [SerializeField] private FloatingMotionController floatingMotion;
    [SerializeField] private float marginRight = 0f;
    [SerializeField] private float marginBottom = 0f;

    /// <summary>現在のメッシュ Bounds に基づいて右下に配置する。</summary>
    public void PositionAtBottomRight();
}
```

## 4. FloatingMotionController（浮遊アニメーション）

### 4.1 責務

VRM ボーン構造に干渉せず、ルート Transform を動かして  
自然な浮遊感と移動モーションを実現します。

### 4.2 浮遊アニメーション（アイドル時）

3 層のサイン波合成による Y 軸・X 軸・Z 軸（傾き）の動き。

| 軸 | 振動数 | 説明 |
|----|--------|------|
| Y（メイン） | 0.55 Hz | 基本の上下 |
| Y（サブ） | 1.10 Hz | 高周波ゆらぎ |
| Y（スロー）| 0.28 Hz | ゆったりした波 |
| X | 0.28 Hz | 横方向のドリフト |
| Z（傾き） | 0.40 Hz | 緩やかな揺れ |

### 4.3 移動アニメーション（3 フェーズ）

```
MoveToAsync(targetPosition, ct)
  Phase 1: Wind-up   (0.25s)  進行方向に 40% ひねり・30% 傾き
  Phase 2: Travel    (0.60s)  目標の 6% オーバーシュート、最大傾き到達
  Phase 3: Settle    (0.35s)  スプリングカーブで目標へ収束
```

### 4.4 インターフェース

```csharp
public class FloatingMotionController : MonoBehaviour
{
    public Vector3 AnchorPosition { get; private set; }

    /// <summary>基点位置を設定する（キャラクター配置後に呼ぶ）。</summary>
    public void SetAnchorPosition(Vector3 position);

    /// <summary>指定座標へアニメーション移動する。</summary>
    public async UniTask MoveToAsync(Vector3 target, CancellationToken ct);
}
```

## 5. CharacterDragController（ドラッグ操作）

### 5.1 責務

マウスによるキャラクターのドラッグ移動を制御します。

### 5.2 操作フロー

```
1. キャラクター上でマウス左ボタン押下を検出
   └─ Raycast でキャラクターの Bounds と交差を確認
2. 長押し判定（0.25 秒）or 移動判定（8 ピクセル以上）でドラッグ開始
   └─ ドラッグ用アニメーション（VRMA）を再生
3. ドラッグ中：マウス位置に追従
4. マウスボタン離し
   └─ FloatingMotionController.MoveToAsync() で最終位置に滑らかに移動
   └─ デフォルトアニメーションに戻す
```

**定数**

| 定数 | 値 | 意味 |
|------|----|------|
| `LongPressDuration` | `0.25f` 秒 | ドラッグ開始の長押し判定時間 |
| `DragMoveThreshold` | `8f` ピクセル | ドラッグ開始の移動量しきい値 |

### 5.3 インターフェース

```csharp
public class CharacterDragController : MonoBehaviour
{
    [SerializeField] private CharacterManager         characterManager;
    [SerializeField] private FloatingMotionController floatingMotion;
    [SerializeField] private VrmaPlayer               vrmaPlayer;
    [SerializeField] private TriggerManager           triggerManager; // クリックトリガー通知用

    // ドラッグ用・デフォルト VRMA プレハブ
    [SerializeField] private GameObject dragVrmaPrefab;
    [SerializeField] private GameObject defaultVrmaPrefab;
}
```

## 6. VrmaPlayer（VRMA 再生）

### 6.1 責務

VRMA（VRM Animation）プレハブを動的にインスタンス化し、キャラクターに適用します。
台本の `＜動作 モーション名＞` は、`motionLibrary` から名前で検索して再生します。

### 6.2 再生処理

```
TryPlayAnimation(motionName)
  1. motionLibrary から一致する motionName を検索
  2. 見つかったら PlayAnimation(vrmaPrefab) を呼ぶ
  3. 見つからなければ false を返す

PlayAnimation(vrmaPrefab)
  1. 既存の VRMA インスタンスを破棄
  2. vrmaPrefab を Instantiate してキャラクターの子にアタッチ
  3. キャラクターの position / rotation / scale をコピー
  4. VrmInstance.Runtime.VrmAnimation に参照を設定
  5. Animation または Animator コンポーネントを自動検出して再生

StopAnimation()
  └─ PlayAnimation(defaultVrmaPrefab) でデフォルトに戻す
```

### 6.3 インターフェース

```csharp
public class VrmaPlayer : MonoBehaviour
{
    [SerializeField] private CharacterManager characterManager;
    [SerializeField] private GameObject       defaultVrmaPrefab;
    [SerializeField] private List<NamedVrmaClip> motionLibrary = new();

    /// <summary>モーション名から対応する VRMA を検索して再生する。</summary>
    public bool TryPlayAnimation(string motionName);

    /// <summary>指定の VRMA プレハブを再生する。</summary>
    public void PlayAnimation(GameObject vrmaPrefab = null);

    /// <summary>デフォルト VRMA に戻す。</summary>
    public void StopAnimation();
}

[Serializable]
public class NamedVrmaClip
{
    public string     motionName;
    public GameObject vrmaPrefab;
}
```
